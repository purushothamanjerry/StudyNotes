01 · Introduction

What is IP Addressing?

An IP address (Internet Protocol address) is a unique numerical label assigned to every device on a network that uses the Internet Protocol for communication. Like a postal address identifies a home, an IP address identifies a device — enabling routers to deliver data packets to the right destination across complex, multi-hop networks.

💡
Real-World Analogy

Think of the internet as a global city. Every device is a building with a unique street address (IP). Data packets are letters. Routers are postal workers reading addresses and forwarding mail through sorting centres (autonomous systems) until it reaches the destination building.

Why IP Addressing Exists

Early ARPANET nodes used a simple hosts.txt flat file listing every known host. By 1983 this approach collapsed — networks grew too fast. RFC 791 (1981) standardized the Internet Protocol with a hierarchical, scalable addressing scheme. The key insight: addresses should encode location information (network prefix), not just identity, so routers can make efficient forwarding decisions without knowing every host.

🎯

Unique Identification

Every connected device gets a globally (or locally) unique address, preventing packets from being mis-delivered.

🗺️

Logical Routing

The network prefix encodes location, letting routers aggregate millions of routes into summaries and scale to the global internet.

📦

Layer 3 Packet Delivery

IP headers in every packet carry source and destination addresses enabling end-to-end delivery across heterogeneous links.

🏗️

Hierarchical Structure

Addresses split into network + host portions, enabling VLSM, route summarisation, and efficient address allocation.

02 · Core Concepts

Network vs Host Portion

Every IP address is logically split into two parts. The Network ID identifies which network the device belongs to. The Host ID identifies the specific device within that network. A subnet mask performs the division.

Subnet Mask Applied — Bitwise AND
IP Address:   192.168.10.25
Subnet Mask:  255.255.255.0  (/24)

Mask (bin):  11111111.11111111.11111111.00000000
IP   (bin):  11000000.10101000.00001010.00011001
             ←───── Network (24 bits) ─────→←Host (8)→

AND result → Network Address: 192.168.10.0
Host portion:                 25
Broadcast:                    192.168.10.255
Usable range:   192.168.10.1  –  192.168.10.254

⚠️
Common Misconception

The IP you see on your laptop (e.g. 192.168.1.x) is a private IP assigned by your router. Your router holds the public IP from your ISP. NAT at the router translates between them. Services on the internet see your public IP only.

03 · IP Classes

Classful Addressing (Historical)

Before CIDR, the first bits of the IP address determined its class, which implied a fixed subnet mask. Classful addressing was used from 1981–1993 and is still tested in exams.

Class Detection — Leading Bit Algorithm
if first_bit == 0:           → Class A  (0xxxxxxx)
elif first_two == "10":     → Class B  (10xxxxxx)
elif first_three == "110":  → Class C  (110xxxxx)
elif first_four == "1110": → Class D  (multicast)
else:                        → Class E  (reserved)

Example: 192.168.1.1
  192 in binary = 11000000
  First 3 bits = 110 → Class C ✓

Example: 10.0.0.1
  10 in binary = 00001010
  First bit = 0 → Class A ✓
🔬
Why Classful Addressing Failed

A company needing 300 hosts was forced to get a Class B address (65,534 hosts), wasting 65,234 addresses. This caused rapid IPv4 exhaustion. CIDR (RFC 1519, 1993) replaced fixed classes with variable-length prefix notation (/n), enabling right-sized allocations.

04 · Subnetting & CIDR

Subnetting & CIDR

Subnetting divides a larger network into smaller logical segments (subnets), reducing broadcast domains, improving security, and enabling efficient address use. CIDR (Classless Inter-Domain Routing, RFC 4632) represents any prefix length as /n, replacing the rigid class system.

Subnet Math — Step by Step

Subnetting Example — 192.168.10.0/25
Prefix length = 25  →  25 network bits, 7 host bits

Subnet Mask (binary):
11111111.11111111.11111111.10000000
Decimal: 255.255.255.128

Host bits = 32 - 25 = 7
Total addresses  = 2^7  = 128
Usable hosts     = 128 - 2 = 126  (net + broadcast removed)

Subnet 1 (block 0):
  Network:   192.168.10.0
  First host:192.168.10.1
  Last host: 192.168.10.126
  Broadcast: 192.168.10.127

Subnet 2 (block 128):
  Network:   192.168.10.128
  First host:192.168.10.129
  Last host: 192.168.10.254
  Broadcast: 192.168.10.255

VLSM — Variable Length Subnet Masking

VLSM lets different subnets within the same block have different prefix lengths. Modern enterprise networks are always VLSM-designed — allocate the largest subnet first, then work down.

VLSM Design — 10.0.0.0/8 (enterprise example)
Requirements: HQ=500 hosts, BranchA=100, BranchB=50, WAN=2, Mgmt=14

Step 1 — HQ (needs ≥500):   10.0.0.0/23    → 510 usable
Step 2 — BranchA (≥100):   10.0.2.0/25    → 126 usable
Step 3 — BranchB (≥50):    10.0.2.128/26  → 62 usable
Step 4 — Mgmt (≥14):       10.0.2.192/28  → 14 usable
Step 5 — WAN link (=2):    10.0.2.208/30  → 2 usable

Total IPs allocated: ~714  |  Wasted: minimal ✓  |  Classful equiv: Class A (16M wasted) ✗
05 · IPv6

IPv6 — The Next Generation Protocol

IPv4 provides ~4.3 billion addresses. IANA exhausted its free pool in February 2011. IPv6 (RFC 8200, originally RFC 2460, 1998) uses 128-bit addresses, providing 3.4 × 10³⁸ unique addresses — more than enough for every grain of sand on Earth to have its own IP.

IPv6 Address Format
Full form (8 groups of 4 hex digits):
2001:0db8:85a3:0000:0000:8a2e:0370:7334
     ↑    ↑    ↑    ↑    ↑    ↑    ↑    ↑
Each group = 16 bits.   Total = 128 bits.

RFC 5952 Compressed form (two rules):
  1. Remove leading zeros per group
  2. Replace longest run of all-zero groups with ::
2001:db8:85a3::8a2e:370:7334  ← compressed

Special addresses:
::1         = loopback (≡ 127.0.0.1)
::          = unspecified / default route
fe80::1/64  = link-local
ff02::1     = all-nodes multicast (≡ broadcast)
ff02::2     = all-routers multicast

IPv4 (RFC 791)

32-bit — ~4.3 billion addresses
Dotted decimal (192.168.1.1)
Manual config or DHCP

Routers fragment packets

NAT required at scale
Broadcast traffic

IPv6 (RFC 8200)

128-bit — 3.4×10³⁸ addresses
Colon-hex (2001:db8::1)
SLAAC auto-configuration
Fixed header exactly 40 bytes

NDP (Neighbor Discovery) replaces ARP

End-to-end without NAT
Multicast replaces broadcast

SLAAC — Stateless Address Autoconfiguration (RFC 4862)

Devices configure their own IPv6 addresses without a DHCP server. The router provides the network prefix; the device generates its own interface identifier.

Step 1
Generate Link-Local Address
Prefix fe80::/10 + Interface ID (EUI-64 from MAC, or random via RFC 7217 privacy extensions).
Step 2
Duplicate Address Detection (DAD)
Sends a Neighbor Solicitation for its own tentative address. No reply within 1 s = address is unique.
Step 3
Router Solicitation (RS)
Host sends RS to ff02::2 (all-routers multicast) requesting network prefix information.
Step 4
Router Advertisement (RA)
Router replies with network prefix (e.g. 2001:db8::/64), default gateway, MTU, and flags.
Step 5
Global Address Formed
Host combines prefix from RA with its interface ID → full global unicast address. No DHCP server needed!

Version (4 bits) — IPv4=4, IPv6=6
▼

IPv4 sets this to 0100 (4). IPv6 uses 0110 (6). The receiver checks this first to know how to parse the rest. Dual-stack hosts process both; routers with IPv4 only drop IPv6 packets silently.

IHL — Internet Header Length (4 bits)
▼

Specifies header length in 32-bit words. Minimum = 5 (5 × 4 bytes = 20 bytes, no options). Maximum = 15 (15 × 4 = 60 bytes). The payload starts at byte offset IHL × 4. Without this field, the OS wouldn't know where the payload begins.

TTL — Time To Live (8 bits) — Routing loop prevention
▼

Each router decrements TTL by 1. At 0: packet dropped, ICMP Type 11 (Time Exceeded) sent back. Defaults: Linux=64, Windows=128, Cisco IOS=255. Traceroute exploits TTL by probing with TTL=1,2,3… collecting ICMP responses to map the route.

Traceroute Mechanism
Probe TTL=1 → Router1 drops → ICMP Time Exceeded → reveals R1 IP
Probe TTL=2 → Router2 drops → ICMP Time Exceeded → reveals R2 IP
Probe TTL=3 → Destination  → ICMP Echo Reply / Port Unreachable

Flags & Fragment Offset — Fragmentation control
▼

Flags (3 bits): Bit 0 = Reserved (0). Bit 1 = DF (Don't Fragment) — if set and packet exceeds link MTU, router drops and sends ICMP "Frag Needed". Bit 2 = MF (More Fragments) — set on all fragments except the last.

Fragmentation Example — 4000-byte packet on 1500-byte MTU link
ID = 0xAB12 for all fragments

Fragment 1: bytes 0–1479    Offset=0    MF=1  Length=1500
Fragment 2: bytes 1480–2959 Offset=185  MF=1  Length=1500
Fragment 3: bytes 2960–3999 Offset=370  MF=0  Length=1040
Offset unit = 8 bytes: 1480/8 = 185 ✓
07 · ARP

ARP — Address Resolution Protocol

IP operates at Layer 3 but physical delivery uses Layer 2 MAC addresses. ARP (RFC 826) bridges this gap by resolving a known IP address to an unknown MAC address on the local network segment.

ARP Flow Diagram

ARP Process — Detailed
1. PC-A wants to send to 192.168.1.20
2. PC-A checks ARP cache: arp -a → no entry found
3. PC-A broadcasts ARP Request:
     Src MAC: aa:bb:cc:11:22:33
     Dst MAC: ff:ff:ff:ff:ff:ff (broadcast)
     Opcode: 1 (request)
     "Who has 192.168.1.20?"
4. All hosts receive it. PC-B recognises its own IP.
5. PC-B sends ARP Reply (unicast):
     Src MAC: dd:ee:ff:44:55:66
     Dst MAC: aa:bb:cc:11:22:33
     Opcode: 2 (reply)
     "192.168.1.20 is at dd:ee:ff:44:55:66"
6. PC-A caches: 192.168.1.20 → dd:ee:ff:44:55:66 (TTL ~2-20 min)
7. PC-A encapsulates IP packet in Ethernet frame to PC-B's MAC

ARP cache TTL: Linux=60s, Windows=2min, Cisco=4h (default)
🚨
ARP Spoofing / ARP Poisoning

An attacker sends gratuitous ARP replies with a fake MAC mapping, poisoning neighbour caches to redirect traffic through their machine (MITM). Mitigated by Dynamic ARP Inspection (DAI) on managed switches — it validates ARP packets against a trusted DHCP snooping binding table.

08 · NAT & PAT

NAT & PAT — Network Address Translation

NAT allows many private hosts to share one (or a few) public IP addresses, conserving the IPv4 address space. Introduced in RFC 1631 (1994) as a temporary measure; it's now permanently embedded in network infrastructure.

Static NAT: One private IP maps permanently to one public IP. Used for inbound servers (web, mail, VPN) that must be reachable from the internet.

Cisco IOS — Static NAT
! Permanent 1:1 mapping
ip nat inside source static 192.168.1.100 203.0.113.10

interface GigabitEthernet0/0
  ip nat inside   ! LAN side
interface GigabitEthernet0/1
  ip nat outside  ! WAN/internet side

Dynamic NAT: A pool of public IPs is shared dynamically. When sessions end, IPs return to the pool. Suitable when you have multiple public IPs but fewer than hosts.

Cisco IOS — Dynamic NAT
ip nat pool PUBPOOL 203.0.113.1 203.0.113.10 netmask 255.255.255.240
access-list 1 permit 192.168.1.0 0.0.0.255
ip nat inside source list 1 pool PUBPOOL

PAT (Port Address Translation) — also called NAT Overload: many private IPs share ONE public IP, differentiated by unique source port numbers. This is what every home router does.

PAT Translation Table (home router)

09 · Routing

IP Routing Fundamentals

Routing is the process of selecting the optimal path for packets to travel from source to destination across multiple interconnected networks. Routers consult their routing table (FIB — Forwarding Information Base) to make per-packet forwarding decisions.

Linux Routing Table — ip route show
default via 192.168.1.1 dev eth0 proto dhcp metric 100
10.0.0.0/8      via 10.10.1.1   dev eth1  metric 200
192.168.1.0/24  dev eth0      proto kernel scope link src 192.168.1.50
172.16.0.0/12   via 192.168.1.254 dev eth0 metric 50

Columns: Destination/Prefix → Next-Hop → Interface → Metric (lower = preferred)
"default" = 0.0.0.0/0 — catches everything not matched more specifically

Longest Prefix Match (LPM)

The fundamental routing algorithm. When multiple routes match a destination, the router selects the one with the longest (most specific) prefix. Implemented in hardware using TCAM for O(1) lookup.

LPM Example
Routing table:
  10.0.0.0/8       via 1.1.1.1   (matches 8 bits)
  10.10.0.0/16     via 2.2.2.2   (matches 16 bits)
  10.10.10.0/24    via 3.3.3.3   (matches 24 bits ← most specific)
  0.0.0.0/0        via 4.4.4.4   (default — 0 bits)

Destination: 10.10.10.50
Matches /8, /16, /24 — Winner: /24 (longest prefix)
→ Forwarded via 3.3.3.3

Destination: 10.99.0.1
Matches /8 only → Forwarded via 1.1.1.1

Destination: 8.8.8.8
No specific match → Default route via 4.4.4.4

12 · Security

IP Security Concepts

🎭

IP Spoofing

Crafting packets with a forged source IP to impersonate another host, bypass ACLs, or amplify DDoS attacks. Foundation of reflection amplification attacks (DNS, NTP, memcached).

☠️

ARP Poisoning

Gratuitous ARP replies poison neighbour caches to redirect LAN traffic through the attacker — a classic Layer 2 MITM. Countered by DAI on managed switches.

💣

Fragmentation Attacks

Teardrop (overlapping offsets crash stacks), Tiny Fragment (first fragment too small for transport header, evades stateless ACLs), Fragment flooding exhausts reassembly buffers.

🔍

ICMP Reconnaissance

Ping sweeps enumerate live hosts. ICMP redirect messages can poison routing tables. Selective ICMP filtering at perimeters limits exposure without breaking PMTUD.

IP Spoofing — DDoS Amplification Attack

Reflection / Amplification Flow
1. Attacker spoofs victim IP (1.2.3.4) as source
2. Sends small queries to thousands of open resolvers:
   Src: 1.2.3.4 (victim)  → DNS server: "ANY isc.org" (40 bytes)
3. DNS servers send large responses TO VICTIM:
   DNS response: ~3000 bytes → 75x amplification factor
4. Victim 1.2.3.4 receives terabits of unsolicited traffic

Defence: BCP38 — ISPs filter packets where source IP
         is not in the originating AS's address space.
Also: rate-limit DNS ANY queries, use response-rate limiting (RRL)

Hardening at Network Edge

uRPF — Unicast Reverse Path Forwarding (anti-spoofing)
▼

Verifies source IP is reachable via the interface the packet arrived on. Strict mode drops if source isn't in the routing table via the same interface. Loose mode drops only if source is completely unreachable.

Cisco — uRPF
interface GigabitEthernet0/1   ! WAN-facing
  ip verify unicast source reachable-via rx  ! strict
  ip verify unicast source reachable-via any  ! loose
Bogon Filtering — block reserved IPs at perimeter
▼

Internet-facing routers should drop inbound packets claiming to originate from RFC 1918 private, loopback, APIPA, or unallocated (bogon) addresses.

Cisco ACL — Bogon Filter
ip access-list extended BOGON-FILTER
  deny ip 10.0.0.0 0.255.255.255 any
  deny ip 172.16.0.0 0.15.255.255 any
  deny ip 192.168.0.0 0.0.255.255 any
  deny ip 127.0.0.0 0.255.255.255 any
  deny ip 169.254.0.0 0.0.255.255 any
  permit ip any any
interface GigabitEthernet0/1
  ip access-group BOGON-FILTER in

14 · Troubleshooting

Troubleshooting Guide

Always follow the OSI model bottom-up. Each step isolates a different layer. Don't skip steps — a wrong assumption wastes time.

Systematic Diagnosis Flowchart

🔍 Cannot reach destination — where to start?
↓
Step 1: ping 127.0.0.1 — tests TCP/IP stack (loopback). Fails = OS/driver issue.
↓
Step 2: ping <own IP> — tests NIC and driver. Fails = NIC or driver issue.
↓
Step 3: ping <default gateway> — tests local LAN/VLAN connectivity. Fails = switch, cable, VLAN, or subnet mask issue.
↓
Step 4: ping 8.8.8.8 — tests internet routing and NAT. Fails = WAN link, default route, or NAT misconfiguration.
↓
Step 5: ping google.com — tests DNS resolution. Fails = DNS server unreachable or misconfigured.
↓
✅ All 5 pass → Problem is application-layer (firewall rules, service down, TLS cert, port blocked)
❌ Fails at step N → Problem is at that exact layer. Fix there first.

Common Issues & Root Causes

Common  Wrong Subnet Mask — can reach some hosts, not others
▼

If your mask is /24 but should be /25, hosts in the upper half appear to be on a different network. Your device sends their packets to the default gateway instead of the local link. ARP requests are never sent for those hosts locally.

Diagnosis
ip addr show eth0            # verify mask is correct
ip route show                # check connected network route
ping 192.168.1.200           # should be local if /24
ip neigh show                # ARP resolving? Or going to GW?
Intermediate  Duplicate IP — intermittent random connectivity drops
▼

Two hosts with the same IP fight over ARP. Each host's ARP reply overwrites the other's cache entries on nearby devices, causing sessions to be silently hijacked by the wrong host.

Detection
arping -D -I eth0 192.168.1.100
# Exit code != 0 = duplicate found
# Wireshark filter: arp.duplicate-address-frame
# Linux dmesg: "ARPING: Duplicate address 192.168.1.x"
Intermediate  PMTUD Black Hole — TCP connects but hangs after handshake
▼

The 3-way handshake works (small packets) but data transfer stalls. A firewall between source and destination silently drops ICMP "Fragmentation Needed" messages, so the sender never knows to reduce packet size.

Detection & Fix
ping -M do -s 1472 target.com  # DF set — fails? = black hole
ping -M do -s 1300 target.com  # smaller size passes?

# Fix 1: Allow ICMP type 3 code 4 through firewalls
iptables -A INPUT -p icmp --icmp-type fragmentation-needed -j ACCEPT

# Fix 2: TCP MSS clamping (clamps SYN/SYN-ACK MSS field)
iptables -A FORWARD -p tcp --tcp-flags SYN,RST SYN \
  -j TCPMSS --clamp-mss-to-pmtu
Advanced  Asymmetric Routing — sessions reset unexpectedly
▼

Outbound traffic takes one path, return traffic takes a different path. Stateful firewalls only tracking sessions on one path reset connections they haven't seen the SYN for. Common in multi-ISP and SD-WAN environments.

Detection
traceroute target.com          # outbound path
tcpdump -i eth0 -n host target.com  # watch which interface handles returns
# If inbound arrives on eth1 but sessions tracked on eth0 = asymmetric
# Fix: use ECMP hash or policy routing to pin flows per-interface
Advanced  NAT Table Exhaustion — new connections refused
▼

PAT tracks connections in a state table. Under DDoS or misconfigured applications opening thousands of half-open TCP connections, the NAT table fills up and new sessions are dropped.

Cisco — Monitor NAT Table
show ip nat translations total
show ip nat statistics
! If "max entries" reached:
ip nat translation max-entries 500000
ip nat translation tcp-timeout 120   ! reduce from 86400s default
ip nat translation udp-timeout 30
15 · Interview Q&A

Interview Questions & Answers

Click any card to reveal the model answer. Levels: Beginner Intermediate Advanced

BeginnerWhat is the difference between a public and private IP?▼
Public IPs are globally unique and routable on the internet, assigned by ISPs from IANA-allocated ranges. Private IPs (RFC 1918: 10.x.x.x, 172.16–31.x.x, 192.168.x.x) are used only within private networks and are not routed on the internet. NAT translates private IPs to public IPs at the network edge (typically your router or firewall).
BeginnerWhat is a subnet mask and how does it work?▼
A subnet mask is a 32-bit number that defines which portion of an IP address is the network ID and which is the host ID. Applied via bitwise AND: IP AND Mask = Network Address. A /24 mask (255.255.255.0) means the first 24 bits identify the network and the last 8 bits identify the host. If destination IP AND mask equals your own network address, deliver directly; otherwise, send to the default gateway.
BeginnerHow many usable hosts does a /27 subnet provide?▼
/27 has 32 - 27 = 5 host bits. 2^5 = 32 total addresses. Minus 2 (network address + broadcast) = 30 usable hosts. Subnet mask = 255.255.255.224. Block size = 32. There are 8 /27 subnets within a /24 (256 ÷ 32 = 8).
IntermediateWhat happens when two hosts on the same LAN have the same IP address?▼
Both hosts respond to ARP requests for that IP. The last ARP reply wins — overwriting neighbour caches across the LAN. Both hosts experience intermittent connectivity as ARP entries flip between their MACs. Linux detects this via gratuitous ARP and logs "ARPING: Duplicate address" to dmesg. Detected with: arping -D -I eth0 <IP>. On managed switches, Dynamic ARP Inspection (DAI) prevents this by validating ARP packets against the DHCP snooping binding table.
IntermediateExplain the purpose of TTL and what happens when it reaches zero.▼
TTL (Time To Live) is an 8-bit field decremented by 1 at each router hop. It prevents packets from looping forever in routing loops. When TTL reaches 0, the router drops the packet and sends ICMP Type 11 (Time Exceeded) back to the source with its own IP address. Traceroute exploits this: it sends probes with TTL=1, 2, 3… and collects the ICMP replies to map the route hop by hop.
IntermediateWhat is PAT and how does it differ from static NAT?▼
Static NAT is a permanent 1:1 mapping between a private and public IP. PAT (Port Address Translation, a.k.a. NAT overload) maps many private IPs to one public IP using unique source port numbers to distinguish sessions. A return packet arrives at the public IP; the router checks the destination port in its NAT table to find which internal host to forward to. PAT is why a home with 20 devices only needs one public IP.
AdvancedExplain Longest Prefix Match and how it's implemented in hardware.▼
LPM selects the route with the most specific (longest) matching prefix. A destination may match multiple prefixes (/8, /16, /24) — the /24 wins because it narrows the address space most. In software, this is done via a trie or Patricia tree. In high-speed routers, TCAM (Ternary Content-Addressable Memory) performs LPM in O(1) using parallel lookup across all entries simultaneously, enabling line-rate forwarding of tens of millions of packets per second regardless of routing table size (currently ~900k BGP routes).
AdvancedWhat is a PMTUD black hole and how do you fix it?▼
PMTUD discovers the smallest MTU along a path by sending DF-flagged packets. If a link can't handle the size, it returns ICMP Type 3 Code 4 "Fragmentation Needed." A black hole occurs when a misconfigured firewall silently drops these ICMP messages. The TCP sender never learns to reduce packet size, so large data transfers stall silently after the 3-way handshake. The SYN/SYN-ACK succeed (small packets) but bulk data hangs. Fixes: (1) Allow ICMP type 3 code 4 through firewalls. (2) TCP MSS clamping at the firewall: iptables TCPMSS --clamp-mss-to-pmtu adjusts the MSS field in SYN packets to match path MTU.
Advanced ScenarioA host can ping 8.8.8.8 but cannot reach 172.16.50.100. What do you check?▼
1. Is 172.16.50.0/24 a private range? Yes — check if there's a route to it (ip route show). 2. If no route, add a static route or check if VPN/MPLS is up. 3. If route exists: traceroute to see where it drops. 4. Check if the destination network's firewall is blocking your source IP. 5. Verify the remote subnet's default gateway can route back to you (asymmetric routing issue). 6. Check MTU — private links sometimes have lower MTU (VPN overhead). 7. If via VPN: check tunnel status, PSK, phase 1/2 SA establishment.
AdvancedHow does IPv6 SLAAC work and why doesn't it need DHCP?▼
SLAAC (RFC 4862) allows hosts to self-configure IPv6 addresses. The host generates a link-local address (fe80::/10 + EUI-64 from MAC, or random per RFC 7217). After DAD confirms uniqueness, it sends a Router Solicitation to ff02::2. The router replies with a Router Advertisement containing the network prefix (e.g., 2001:db8::/64). The host appends its interface ID to form a global unicast address. No central DHCP server is needed because the prefix comes from the router and the host ID is self-generated. DHCPv6 is still optionally used for DNS server addresses and other options not carried in RA.

17 · Summary

Summary & Key Takeaways

🌐

IP is Logical, Hierarchical Addressing

IP addresses encode both network location (prefix) and host identity, enabling routers to aggregate millions of routes and scale to the global internet without knowing every host.

🔢

CIDR & VLSM Solved Exhaustion

Classful addressing wasted billions of addresses. CIDR allows any prefix length; VLSM lets you right-size each subnet. Always allocate largest subnets first when designing a VLSM scheme.

🚀

IPv6 is Inevitable

3.4×10³⁸ addresses, fixed 40-byte header, built-in IPSec, SLAAC auto-configuration, and NDP replacing ARP. Dual-stack is the migration strategy; native IPv6 is the destination.

⚙️

Headers Carry Critical Metadata

TTL prevents loops, Protocol enables demux, DF flag controls fragmentation, DSCP enables QoS, and Identification ties fragments together. Every field exists for a reason.

🔒

IP Security is Multi-Layered

BCP38 stops spoofing at ISP borders; uRPF on routers; DAI on switches stops ARP poisoning; bogon filters drop reserved ranges; ICMP filtering protects without breaking PMTUD.

🛠️

Troubleshoot Bottom-Up, Always

loopback → own IP → gateway → public IP → hostname. Each step isolates a layer. Never guess — run the commands. Traceroute, tcpdump, and ip neigh are your best friends.
