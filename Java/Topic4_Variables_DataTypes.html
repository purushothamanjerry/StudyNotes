What is a Variable?
Definition: A variable is a named memory location that stores a value. It has three parts: a data type, a name, and a value. 
// Syntax: dataType variableName = value;
int age = 25;           // type=int, name=age, value=25
String name = "Alice"; // type=String, name=name, value="Alice"
boolean isActive = true; // type=boolean, name=isActive, value=true

// Declaration without initialization
int x;       // declared but not initialized
x = 10;     // initialized later
⚠️ Local variables MUST be initialized before use:
int x; System.out.println(x); ❌ → Compile error: "variable x might not have been initialized"
Instance variables get default values automatically (0, false, null).

3 Types of Variables in Java

Local Variable

Declared inside a method or block. Only accessible within that method. No default value — must initialize before use. Stored on Stack.

Instance Variable

Declared inside a class but outside methods. Each object has its own copy. Has default values. Stored on Heap inside the object.

Static Variable

Declared with static keyword. Shared across ALL objects of the class — only one copy exists. Stored in Method Area.

public class Employee {

    // Instance variable — each Employee object has its own
    String name;        // default: null
    int salary;         // default: 0

    // Static variable — shared across ALL Employee objects
    static String company = "TechCorp";
    static int count = 0;

    public void display() {
        // Local variable — only exists inside this method
        int bonus = 5000;
        System.out.println(name + " bonus: " + bonus);
    }
}

Employee e1 = new Employee();
Employee e2 = new Employee();
e1.name = "Alice";   // only e1's name changes
Employee.company = "NewCorp"; // changes for ALL employees!

Variable Scope

Scope defines where a variable can be accessed in the code. A variable lives from where it is declared to the closing brace of its block.

public class ScopeDemo {
    int instanceVar = 10;   // class scope — whole class

    public void method() {
        int a = 20;            // method scope — whole method

        if (true) {
            int b = 30;        // block scope — only inside if
            System.out.println(b);   // ✅ OK
            System.out.println(a);   // ✅ OK
        }

        System.out.println(a);       // ✅ OK
        // System.out.println(b);   ❌ Error — b out of scope
    }
}
⚠️ Variable Shadowing:
A local variable with the same name as an instance variable shadows it inside the method. Use this.variableName to refer to the instance variable. 
int age = 30;  // instance variable
public void setAge(int age) {  // parameter shadows instance var
    this.age = age;  // this.age = instance, age = parameter
}

Type Casting — Widening & Narrowing
Type Casting = converting one data type to another. Two types: Widening (automatic, safe) and Narrowing (manual, may lose data).

Widening Casting (Implicit) — automatic

Converting smaller type → larger type. No data loss. Java does it automatically.

byte → short → int → long → float → double 

int i = 100;
long l = i;      // automatic widening — int to long ✅
double d = i;    // automatic widening — int to double ✅
System.out.println(d);  // 100.0

Narrowing Casting (Explicit) — manual

Converting larger type → smaller type. May lose data. Must do manually with cast operator.

double d = 9.99;
int i = (int) d;   // explicit narrowing — truncates decimal
System.out.println(i);  // 9 (not 10! decimal is cut, not rounded)

int big = 300;
byte b = (byte) big;  // 300 doesn't fit in byte (-128 to 127)
System.out.println(b);  // 44 (overflow! data lost)
⚠️ Narrowing does NOT round — it truncates!
(int) 9.99 → 9 not 10
(int) -9.99 → -9 not -10

String Conversions

// int → String
int n = 42;
String s1 = String.valueOf(n);   // "42"
String s2 = "" + n;              // "42" (concatenation trick)
String s3 = Integer.toString(n); // "42"

// String → int
String s = "42";
int x = Integer.parseInt(s);    // 42
int y = Integer.valueOf(s);     // 42 (returns Integer, autoboxed)

var Keyword (Java 10+)

The var keyword enables local variable type inference — the compiler automatically infers the type from the value assigned. Java is still strongly typed — var is just syntactic sugar.

// Without var
ArrayList<String> list = new ArrayList<String>();

// With var — cleaner!
var list = new ArrayList<String>();  // compiler infers ArrayList<String>

var name = "Alice";   // inferred as String
var age  = 25;        // inferred as int
var pi   = 3.14;      // inferred as double
⚠️ var restrictions:
❌ Cannot use var for instance variables or method parameters
❌ Cannot use var without initialization: var x; → Error
❌ Cannot assign null: var x = null; → Error (type unknown)
✅ Only for local variables inside methods

Naming Rules & Conventions

Rules (must follow — compiler enforces)

  • Start with letter, _, or $ — NOT a digit
  • Can contain letters, digits, _, $
  • Cannot use Java keywords (int, class, for...)
  • Case sensitive — age and Age are different
  • No spaces allowed

Conventions (should follow — best practice)

  • Variables & methods → camelCase: firstName, getAge()
  • Classes → PascalCase: StudentDetails
  • Constants → UPPER_SNAKE_CASE: MAX_SIZE
  • Packages → lowercase: com.company.project
  • Use meaningful names: age not a
// Valid variable names
int age = 25;
int _count = 0;
int $price = 100;
int myAge2 = 30;

// Invalid variable names
// int 2age = 25;     ❌ starts with digit
// int my age = 25;   ❌ space not allowed
// int int = 25;      ❌ keyword not allowed
// int my-age = 25;   ❌ hyphen not allowed

Constants — final keyword

A constant is a variable whose value cannot be changed after initialization. Use the final keyword.

final int MAX_SIZE = 100;
// MAX_SIZE = 200;  ❌ Error — cannot reassign final variable

// Convention: constants use UPPER_SNAKE_CASE
final double PI = 3.14159;
final String APP_NAME = "CodeTrack";

// Static final = class-level constant (most common)
public static final int MAX_USERS = 1000;
💡 final vs static final:
final — each object has its own constant copy
static final — one constant shared across all objects (preferred for true constants)

Output — print, println, printf
// println — prints and adds new line
System.out.println("Hello");    // Hello↵
System.out.println(42);         // 42↵

// print — prints WITHOUT new line
System.out.print("Hello ");
System.out.print("World");     // Hello World (same line)

// printf — formatted output (like C)
System.out.printf("Name: %s, Age: %d%n", "Alice", 25);
// Name: Alice, Age: 25

// String.format — returns formatted string
String msg = String.format("Score: %.2f", 95.5678);
System.out.println(msg);  // Score: 95.57

Top Interview Questions & Model Answers
Q1. What are the 3 types of variables in Java?
Local variables — declared inside methods, no default value, stored on Stack.
Instance variables — declared in class body, each object has its own copy, default values, stored on Heap.
Static variables — declared with static keyword, shared across all objects, only one copy per class, stored in Method Area.
Q2. What is the difference between widening and narrowing casting?
Widening (implicit) converts a smaller type to a larger type automatically — no data loss. Example: int → long. Narrowing (explicit) converts a larger type to a smaller type manually using cast operator — may lose data. Example: (int) 9.99 → 9 (decimal truncated, not rounded).
Q3. What is the output? int x; System.out.println(x);
Compile error — "variable x might not have been initialized." Local variables in Java do not get default values. You must explicitly initialize them before use. Only instance and static variables get default values.
Q4. What is var in Java? When can you use it?
var is a local variable type inference keyword introduced in Java 10. The compiler infers the type from the assigned value. It can only be used for local variables inside methods — not for instance variables, method parameters, or return types. Java remains strongly typed — var is just syntactic sugar.
Q5. What is the difference between instance variable and static variable?
An instance variable has a separate copy for each object — changing it in one object doesn't affect others. A static variable has only one copy shared across all objects of the class — changing it via one object affects all. Static variables are accessed using the class name, not object reference.
Q6. What is the output? double d = 9.99; int i = (int) d; System.out.println(i);
9 — Narrowing casting truncates the decimal part, it does NOT round. So 9.99 becomes 9, not 10.
Q7. Can you change the value of a final variable?
No. A final variable can only be assigned once. Attempting to reassign it causes a compile-time error. For class-level constants, we use public static final — one constant shared across all objects, accessed via class name.
