Loop Overview — All 4 Types
What is a Loop? A loop repeats a block of code multiple times until a condition becomes false. Java has 4 types of loops.
for loop — use when you know exactly how many times to repeat
while loop — use when you don't know how many times, check condition BEFORE each iteration
do-while loop — runs AT LEAST ONCE, check condition AFTER each iteration
for-each loop — use to iterate over arrays or collections, no index needed
2
for Loop
Syntax: for (initialization; condition; update) { body }
3 parts: init runs once → condition checked → body runs → update → condition checked again... 
// Basic for loop
for (int i = 0; i < 5; i++) {
    System.out.println(i);  // prints 0,1,2,3,4
}

// Count down
for (int i = 5; i > 0; i--) {
    System.out.println(i);  // prints 5,4,3,2,1
}

// Step by 2
for (int i = 0; i <= 10; i += 2) {
    System.out.print(i + " ");  // 0 2 4 6 8 10
}

// Sum of 1 to 10
int sum = 0;
for (int i = 1; i <= 10; i++) {
    sum += i;
}
System.out.println(sum);  // 55

for loop — all parts are optional!

// All parts optional — but semicolons are required
int i = 0;
for ( ; i < 5 ; ) {   // init and update outside
    System.out.println(i);
    i++;
}

// Multiple variables in for loop
for (int a = 0, b = 10; a < b; a++, b--) {
    System.out.println(a + " " + b);
}
3
while Loop
Syntax: while (condition) { body }
Checks condition BEFORE executing body. If condition is false from the start, body never runs (0 times). 
// Basic while loop
int i = 1;
while (i <= 5) {
    System.out.println(i);  // 1,2,3,4,5
    i++;
}

// Read user input until valid (typical use case)
Scanner sc = new Scanner(System.in);
int num = -1;
while (num < 0) {
    System.out.print("Enter positive number: ");
    num = sc.nextInt();
}

// Condition false from start — body never runs
int x = 10;
while (x < 5) {
    System.out.println("Never prints");  // 0 times
}
⚠️ Most common mistake — forgetting to update the variable: 
int i = 0;
while (i < 5) {
    System.out.println(i);
    // forgot i++ → INFINITE LOOP!
}
4
do-while Loop
Syntax: do { body } while (condition);
Executes body FIRST, then checks condition. Runs AT LEAST ONCE — even if condition is false! 
// Basic do-while
int i = 1;
do {
    System.out.println(i);  // 1,2,3,4,5
    i++;
} while (i <= 5);

// Condition false from start — runs ONCE!
int x = 10;
do {
    System.out.println("Runs once!");  // prints this!
} while (x < 5);   // condition false but body already ran

// Classic use case — menu driven program
int choice;
do {
    System.out.println("1. Add  2. Delete  3. Exit");
    choice = sc.nextInt();
    // process choice...
} while (choice != 3);  // repeat until Exit chosen

while loop

Check condition → execute body

May run 0 times if condition false initially

do-while loop

Execute body → check condition

Always runs at least 1 time

5
for-each Loop (Enhanced for)
Syntax: for (dataType variable : arrayOrCollection) { body }
Cleanest way to iterate — no index, no condition, no update needed. 
// Iterating array
int[] numbers = {10, 20, 30, 40, 50};
for (int num : numbers) {
    System.out.println(num);  // 10, 20, 30, 40, 50
}

// Iterating String array
String[] fruits = {"Apple", "Banana", "Cherry"};
for (String fruit : fruits) {
    System.out.println(fruit);
}

// Iterating List (Collections)
List<Integer> list = Arrays.asList(1, 2, 3, 4, 5);
for (int n : list) {
    System.out.print(n + " ");  // 1 2 3 4 5
}
⚠️ Limitations of for-each:
❌ Cannot modify the array/collection while iterating
❌ Cannot access index (use regular for if you need index)
❌ Cannot iterate in reverse
❌ Cannot skip elements easily
✅ Best for read-only iteration over all elements
6
break Statement

break immediately exits the loop — skips remaining iterations and remaining code in the loop body.

// Stop when number found
int[] arr = {3, 7, 2, 9, 5};
for (int num : arr) {
    if (num == 9) {
        System.out.println("Found 9!");
        break;   // exits loop immediately
    }
    System.out.println(num);  // prints 3, 7, 2 (stops before 9)
}

// break in nested loops — only breaks INNER loop
for (int i = 1; i <= 3; i++) {
    for (int j = 1; j <= 3; j++) {
        if (j == 2) break;   // breaks inner loop only
        System.out.println(i + "," + j);
    }
}
// Output: 1,1  2,1  3,1

// Labeled break — break outer loop from inner
outer:
for (int i = 1; i <= 3; i++) {
    for (int j = 1; j <= 3; j++) {
        if (i == 2 && j == 2) break outer;  // breaks OUTER loop
        System.out.println(i + "," + j);
    }
}
// Output: 1,1  1,2  1,3  2,1  (stops at 2,2)
7
continue Statement

continue skips the current iteration and moves to the next — does NOT exit the loop.

// Skip even numbers — print only odd
for (int i = 1; i <= 10; i++) {
    if (i % 2 == 0) continue;  // skip evens
    System.out.print(i + " ");   // 1 3 5 7 9
}

// Skip a specific value
for (int i = 1; i <= 5; i++) {
    if (i == 3) continue;
    System.out.print(i + " ");  // 1 2 4 5
}

break

Exits the loop completely

Remaining iterations are cancelled

Use when: target found, stop condition met

continue

Skips current iteration only

Loop continues with next iteration

Use when: skip certain values, filter elements

8
Nested Loops

A loop inside another loop. Inner loop completes all its iterations for each single iteration of the outer loop.

// Multiplication table
for (int i = 1; i <= 3; i++) {
    for (int j = 1; j <= 3; j++) {
        System.out.print(i * j + "\t");
    }
    System.out.println();
}
// Output:
// 1  2  3
// 2  4  6
// 3  6  9

// Star pattern — classic interview question!
for (int i = 1; i <= 5; i++) {
    for (int j = 1; j <= i; j++) {
        System.out.print("* ");
    }
    System.out.println();
}
// Output:
// *
// * *
// * * *
// * * * *
// * * * * *
Total iterations = outer × inner. A 3×3 nested loop runs 9 times total. A 10×10 loop runs 100 times. Be careful with large nested loops — they can be slow!
9
Infinite Loops

A loop that never stops because the condition never becomes false. Sometimes intentional (servers, game loops), often a bug.

// Intentional infinite loop patterns
while (true) {
    // server listening loop
    // use break to exit when needed
}

for (;;) {
    // infinite for loop (all 3 parts empty)
}

// Common infinite loop bugs
int i = 0;
while (i < 10) {
    System.out.println(i);
    // BUG: forgot i++ → runs forever!
}

for (int j = 0; j < 10; j--) {
    // BUG: j-- instead of j++ → counts to negative infinity!
}

Comparison — When to Use Which Loop
LoopWhen to useRuns 0 times?Index available?
for Know exact number of iterations Yes (if condition false) Yes
while Unknown iterations, check before running Yes (if condition false) Only if you manage it
do-while Must run at least once (menus, prompts) No — always runs once Only if you manage it
for-each Read-only iteration over array/collection Yes (if empty) No
💡 Interview memory trick:
for → "I know how many times" (counter-controlled)
while → "I don't know, check first" (condition-controlled)
do-while → "Run first, ask questions later" (at least once)
for-each → "Just give me each element" (collection traversal)

Top Interview Questions & Model Answers
Q1. What is the difference between while and do-while loop?
In a while loop, the condition is checked BEFORE executing the body — so if the condition is false initially, the body never runs (0 executions). In a do-while loop, the body executes FIRST, then the condition is checked — so it always runs AT LEAST ONCE even if the condition is false. Use do-while for menu-driven programs where you need to show the menu at least once.
Q2. What is the difference between break and continue?
break completely exits the loop — remaining iterations are cancelled. continue skips only the current iteration and moves to the next one — the loop continues. Use break when you want to stop the loop early (e.g., element found). Use continue when you want to skip certain elements (e.g., skip even numbers).
Q3. What is the output? 
for (int i = 0; i < 5; i++) {
    if (i == 3) break;
    System.out.print(i + " ");
}
0 1 2 — loop runs for i=0,1,2 normally. When i=3, break exits the loop immediately. i=4 never runs.
Q4. What is the output? 
for (int i = 0; i < 5; i++) {
    if (i == 3) continue;
    System.out.print(i + " ");
}
0 1 2 4 — loop runs normally, but when i=3, continue skips that iteration. i=4 still runs. 3 is skipped but the loop does not stop.
Q5. What are the limitations of the for-each loop?
for-each cannot access the index, cannot iterate in reverse, cannot modify the collection while iterating (ConcurrentModificationException), and cannot skip to a specific index. It is only suitable for forward, read-only iteration over all elements. Use a regular for loop when you need the index or need to modify elements.
Q6. How do you write an infinite loop in Java?
Three ways: while(true) {}, for(;;) {}, or do {} while(true);. Infinite loops are used intentionally in server processes, game loops, or event listeners where the program should keep running. Use break with a condition to exit when needed.
Q7. What is a labeled break? When would you use it?
A labeled break allows you to break out of an outer loop from inside a nested loop. You add a label before the outer loop (e.g., outer:) and use break outer; inside the inner loop. This is useful when searching a 2D array — once you find what you're looking for, you want to exit both loops, not just the inner one.
