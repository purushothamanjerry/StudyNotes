What is a Class?
Definition: A class is a blueprint/template for creating objects. It defines the properties (fields) and behaviors (methods) that all objects of that type will have. A class is NOT an object — it is the design.

Real World Analogy

Class = Blueprint of a house

Object = Actual house built from blueprint

One blueprint → many houses

One class → many objects

Class contains

  • Fields (instance variables) — state
  • Methods — behavior
  • Constructors — object creation
  • Nested classes — optional
  • Static members — class-level
2
What is an Object?
Definition: An object is an instance of a class. It is a real entity in memory created from the class blueprint. Each object has its own copy of instance variables.

3 Characteristics of an Object

  • State — data stored in fields (name, age, color)
  • Behavior — actions defined by methods (eat, bark, drive)
  • Identity — unique memory address that distinguishes it from other objects
// Class = blueprint
class Dog {
    String name;   // state
    int age;        // state

    void bark() {   // behavior
        System.out.println(name + " says: Woof!");
    }
}

// Objects = instances (each has own state)
Dog dog1 = new Dog();
dog1.name = "Bruno";
dog1.age  = 3;

Dog dog2 = new Dog();
dog2.name = "Max";
dog2.age  = 5;

dog1.bark();  // Bruno says: Woof!
dog2.bark();  // Max says: Woof!
// dog1 and dog2 are independent — different state, same behavior

Class Anatomy — Full Structure
public class Student {

    // ── Static variable (shared by all objects) ──
    static int totalStudents = 0;

    // ── Instance variables (each object has own copy) ──
    private String name;
    private int age;
    private double gpa;

    // ── Constructor ──
    public Student(String name, int age, double gpa) {
        this.name = name;
        this.age  = age;
        this.gpa  = gpa;
        totalStudents++;   // increment shared counter
    }

    // ── Getters & Setters ──
    public String getName()  { return name; }
    public int    getAge()   { return age;  }
    public void   setAge(int age) {
        if (age > 0) this.age = age;
    }

    // ── Instance method ──
    public void display() {
        System.out.println(name + " | Age: " + age + " | GPA: " + gpa);
    }

    // ── Static method ──
    public static int getTotalStudents() {
        return totalStudents;
    }

    // ── toString override ──
    @Override
    public String toString() {
        return "Student{" + name + ", " + age + "}";
    }
}

// Usage
Student s1 = new Student("Alice", 20, 3.8);
Student s2 = new Student("Bob",   22, 3.5);
s1.display();
System.out.println(Student.getTotalStudents());  // 2

new Keyword — Object Creation
new does 3 things:
1. Allocates memory on the Heap for the new object
2. Calls the constructor to initialize the object
3. Returns a reference (memory address) to the new object 
// Dog d = new Dog();
// │         │
// │         └── new allocates memory on Heap + calls constructor
// └── d stores the REFERENCE (address) on Stack

Dog d1 = new Dog();
Dog d2 = d1;   // d2 copies the REFERENCE — both point to same object!

d1.name = "Bruno";
System.out.println(d2.name);  // Bruno — same object!

d1 = null;   // d1 no longer points to object
              // BUT d2 still does — object NOT garbage collected yet
⚠️ Copying reference vs copying object:
Dog d2 = d1; — copies the reference, NOT the object. Both variables point to the SAME object. Changes through d2 affect d1 too!

this Keyword — 5 Uses

this refers to the current object — the object on which the method or constructor is being called.

public class Person {
    String name;
    int age;

    // USE 1 — Resolve ambiguity (parameter shadows instance variable)
    public Person(String name, int age) {
        this.name = name;  // this.name = instance var, name = parameter
        this.age  = age;
    }

    // USE 2 — Call another method of same class
    public void show() {
        this.display();   // same as display() but explicit
    }

    // USE 3 — Call another constructor (constructor chaining)
    public Person(String name) {
        this(name, 0);  // calls Person(String, int) constructor
                         // MUST be first statement!
    }

    // USE 4 — Return current object (method chaining)
    public Person setName(String name) {
        this.name = name;
        return this;   // returns current object
    }

    // USE 5 — Pass current object as argument
    public void register() {
        Database.save(this);  // pass current Person object
    }

    public void display() {
        System.out.println(name + " " + age);
    }
}

// Method chaining using this
Person p = new Person("Alice", 25);
p.setName("Bob").display();  // Bob 25

super Keyword — 3 Uses

super refers to the parent class. Used inside a child class to access parent's members.

class Animal {
    String type = "Animal";

    public Animal(String type) {
        this.type = type;
        System.out.println("Animal created");
    }

    public void sound() {
        System.out.println("Some sound");
    }
}

class Dog extends Animal {
    String type = "Dog";   // shadows Animal's type

    // USE 1 — Call parent constructor (must be first line!)
    public Dog() {
        super("Canine");   // calls Animal(String) constructor
        System.out.println("Dog created");
    }

    // USE 2 — Access parent class variable
    public void showTypes() {
        System.out.println(super.type);  // Canine (parent's)
        System.out.println(this.type);   // Dog (current)
    }

    // USE 3 — Call parent class method
    @Override
    public void sound() {
        super.sound();          // calls Animal's sound()
        System.out.println("Woof!");   // adds Dog's sound
    }
}

Dog d = new Dog();
// Output: Animal created → Dog created
d.sound();
// Output: Some sound → Woof!

toString() Method

Default toString() returns ClassName@hashCode — not useful. Always override it!

// Without override
Dog d = new Dog();
System.out.println(d);  // Dog@6d06d69c ← ugly!

// With override
class Dog {
    String name;
    int age;

    @Override
    public String toString() {
        return "Dog{name='" + name + "', age=" + age + "}";
    }
}

Dog d = new Dog();
d.name = "Bruno"; d.age = 3;
System.out.println(d);        // Dog{name='Bruno', age=3}
System.out.println(d.toString()); // same — println calls toString automatically
💡 toString() is called automatically by:
System.out.println(obj) · String concatenation "" + obj · Debugger display
9
equals() and hashCode()
// Default equals() — compares REFERENCES (like ==)
Dog d1 = new Dog("Bruno");
Dog d2 = new Dog("Bruno");
System.out.println(d1.equals(d2));  // false (different objects in memory)
System.out.println(d1 == d2);       // false

// Override equals() — compare by CONTENT
class Dog {
    String name;

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof Dog)) return false;
        Dog other = (Dog) o;
        return this.name.equals(other.name);
    }

    @Override
    public int hashCode() {
        return name.hashCode();  // must override with equals!
    }
}

Dog d1 = new Dog("Bruno");
Dog d2 = new Dog("Bruno");
System.out.println(d1.equals(d2));  // true (same name)
⚠️ Contract: If you override equals(), you MUST override hashCode() too!
Rule: If a.equals(b) is true → a.hashCode() == b.hashCode() must also be true.
Breaking this contract causes bugs in HashMap, HashSet, and other hash-based collections.

Top Interview Questions & Model Answers
Q1. What is the difference between a class and an object?
A class is a blueprint or template that defines the structure and behavior. An object is an actual instance created from that blueprint. Class exists at compile time; objects exist at runtime in heap memory. One class can have many objects, each with their own state. Example: Dog is a class; new Dog() creates an object.
Q2. What are the uses of the this keyword?
this has 5 uses: (1) Resolve ambiguity between instance variable and parameter with same name. (2) Call another method of the same class. (3) Call another constructor of the same class using this() — must be first statement. (4) Return the current object for method chaining. (5) Pass the current object as an argument to another method.
Q3. What is the difference between == and equals() for objects?
== compares references — checks if both variables point to the exact same memory address (same object). equals() compares content/logical equality — by default same as ==, but can be overridden to compare field values. Always override equals() when you want to compare objects by their data, and always override hashCode() along with it.
Q4. What does the new keyword do?
The new keyword does three things: (1) Allocates memory on the Heap for the new object. (2) Calls the constructor to initialize the object's fields. (3) Returns a reference (memory address) to the newly created object, which is stored in the reference variable on the Stack.
Q5. What is the Object class? Why is it important?
Object is the root class of all Java classes — every class implicitly extends it if no other superclass is specified. It provides fundamental methods like toString(), equals(), hashCode(), getClass(), clone(), and thread methods (wait, notify). This ensures all Java objects have a common interface.
Q6. Why should you override toString()?
The default toString() from Object returns ClassName@hashCode which is meaningless for debugging. Overriding it to return meaningful field values makes debugging easier, makes logging more readable, and is automatically called by System.out.println() and string concatenation.
Q7. Why must you override hashCode() when you override equals()?
The contract between equals() and hashCode() states: if two objects are equal (equals() returns true), they must have the same hash code. Hash-based collections like HashMap and HashSet use hashCode() first to find the bucket, then equals() to find the exact entry. If you override equals() without hashCode(), equal objects may have different hash codes, causing these collections to malfunction — they won't find objects that you know are equal.
