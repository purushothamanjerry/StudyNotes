Lambda Expressions (Java 8)
Lambda = a short, anonymous function. It is the implementation of a functional interface. Eliminates the need for verbose anonymous class syntax.
Syntax: (parameters) -> expression or (parameters) -> { body } 
// Before Java 8 — anonymous class (verbose)
Runnable r1 = new Runnable() {
    public void run() {
        System.out.println("Running...");
    }
};

// Java 8+ — lambda (concise)
Runnable r2 = () -> System.out.println("Running...");

// Lambda syntax variations
// No params
Runnable   run    = () -> System.out.println("Hi");

// One param — parentheses optional
Consumer<String>  print  = s -> System.out.println(s);
Consumer<String>  print2 = (s) -> System.out.println(s);  // same

// Two params
Comparator<Integer> comp = (a, b) -> a - b;

// Multi-line body — use braces and return
Comparator<String> byLen = (a, b) -> {
    if (a.length() != b.length()) return a.length() - b.length();
    return a.compareTo(b);
};

// Lambda with collections — very common!
List<String> names = Arrays.asList("Charlie", "Alice", "Bob");
names.sort((a, b) -> a.compareTo(b));          // sort alphabetically
names.forEach(name -> System.out.println(name)); // print each

InterfaceMethodInput → OutputLambda Example
Predicate<T>test(T)T → booleanx -> x > 0
Function<T,R>apply(T)T → Rs -> s.length()
Consumer<T>accept(T)T → voidx -> System.out.println(x)
Supplier<T>get()nothing → T() -> "Hello"
BiFunction<T,U,R>apply(T,U)T,U → R(a,b) -> a+b

Predicate<Integer> isEven = n -> n % 2 == 0;
System.out.println(isEven.test(4));   // true
System.out.println(isEven.test(3));   // false

// Predicate composition
Predicate<Integer> isPositive = n -> n > 0;
Predicate<Integer> isEvenAndPositive = isEven.and(isPositive);
Predicate<Integer> isEvenOrPositive  = isEven.or(isPositive);
Predicate<Integer> isOdd             = isEven.negate();

Function<String, Integer> strLen = String::length;
Function<Integer, Integer> doubleIt = x -> x * 2;
// Function composition
Function<String, Integer> lenThenDouble = strLen.andThen(doubleIt);
System.out.println(lenThenDouble.apply("Hello"));  // 10 (len=5, 5*2=10)

Method References
Method reference = a shorthand for a lambda that just calls a single method. Syntax: ClassName::methodName
TypeSyntaxLambda equivalent
Static methodClassName::staticMethodx -> ClassName.staticMethod(x)
Instance method (specific obj)obj::instanceMethodx -> obj.instanceMethod(x)
Instance method (any obj)ClassName::instanceMethodx -> x.instanceMethod()
ConstructorClassName::newargs -> new ClassName(args)
List<String> names = Arrays.asList("Alice", "Bob", "Charlie");

// Instance method of any object — String::toUpperCase
names.stream()
     .map(String::toUpperCase)         // same as s -> s.toUpperCase()
     .forEach(System.out::println);    // same as s -> System.out.println(s)

// Static method reference
List<String> nums = Arrays.asList("3", "1", "2");
nums.stream()
    .map(Integer::parseInt)             // static method: Integer.parseInt(s)
    .sorted()
    .forEach(System.out::println);

// Constructor reference
Supplier<ArrayList> listFactory = ArrayList::new;
ArrayList newList = listFactory.get();

Stream API (Java 8)
Stream = a pipeline for processing collections of data in a functional style. Streams are lazy — intermediate operations are not executed until a terminal operation is called. Streams do NOT modify the original collection.

Stream Pipeline

Source → Intermediate Operations (lazy) → Terminal Operation (triggers execution) 

List<Integer> numbers = Arrays.asList(5, 3, 8, 1, 9, 2, 7, 4, 6);

// Classic example pipeline
List<Integer> result = numbers.stream()
    .filter(n -> n > 3)          // keep: 5,8,9,7,4,6
    .sorted()                      // sort: 4,5,6,7,8,9
    .limit(4)                     // take first 4: 4,5,6,7
    .collect(Collectors.toList()); // terminal — collect to list
System.out.println(result);  // [4, 5, 6, 7]

Intermediate Operations (return Stream — lazy)

List<String> names = Arrays.asList("Alice", "Bob", "Charlie", "Alice", "Dave");

names.stream().filter(s -> s.length() > 3);       // filter elements
names.stream().map(String::toUpperCase);           // transform each element
names.stream().sorted();                             // natural sort
names.stream().sorted(Comparator.reverseOrder());  // custom sort
names.stream().distinct();                           // remove duplicates
names.stream().limit(3);                            // take first 3
names.stream().skip(2);                             // skip first 2
names.stream().peek(System.out::println);            // debug — doesn't consume
names.stream().flatMap(s -> Arrays.stream(s.split(""))); // flatten nested

Terminal Operations (trigger execution)

List<Integer> nums = Arrays.asList(1, 2, 3, 4, 5);

// Collect to collections
List<Integer> list    = nums.stream().collect(Collectors.toList());
Set<Integer>  set     = nums.stream().collect(Collectors.toSet());
String       joined  = Arrays.asList("a","b","c").stream()
                           .collect(Collectors.joining(", "));  // "a, b, c"

// Reduce operations
int sum  = nums.stream().mapToInt(Integer::intValue).sum(); // 15
int max  = nums.stream().mapToInt(Integer::intValue).max().getAsInt(); // 5
double avg = nums.stream().mapToInt(Integer::intValue).average().getAsDouble(); // 3.0

// Custom reduce
int product = nums.stream().reduce(1, (a, b) -> a * b); // 120 (1*2*3*4*5)

// Count, any/all/none match, findFirst/Any
long count    = nums.stream().filter(n -> n > 3).count();  // 2
boolean anyPos = nums.stream().anyMatch(n -> n > 4);  // true (5 > 4)
boolean allPos = nums.stream().allMatch(n -> n > 0);  // true
boolean noneNeg = nums.stream().noneMatch(n -> n < 0); // true
Optional<Integer> first = nums.stream().findFirst();    // Optional[1]

// Grouping (very powerful!)
Map<Integer, List<String>> byLength = names.stream()
    .collect(Collectors.groupingBy(String::length));
// {3=[Bob], 4=[Dave], 5=[Alice, Alice], 7=[Charlie]}

// Counting per group
Map<Integer, Long> countByLength = names.stream()
    .collect(Collectors.groupingBy(String::length, Collectors.counting()));
💡 Parallel Streams — easy parallelism:
list.parallelStream().filter(...).collect(...)
Automatically uses multiple CPU cores. Good for large datasets. Not always faster — overhead for small collections.

Optional (Java 8)
Optional<T> is a container that may or may not hold a value. It is a better way to handle null — forces you to explicitly deal with the case where a value might be absent, avoiding NullPointerException. 
// Create Optional
Optional<String> present = Optional.of("Hello");     // has value
Optional<String> empty   = Optional.empty();           // no value
Optional<String> maybe   = Optional.ofNullable(null);  // null → empty
Optional<String> maybe2  = Optional.ofNullable("Hi");  // "Hi" → present

// Check and get
present.isPresent();              // true
empty.isPresent();                // false
empty.isEmpty();                  // true (Java 11)
present.get();                    // "Hello" — throws if empty!

// Safe access
present.orElse("default");      // "Hello" (has value)
empty.orElse("default");         // "default"
empty.orElseGet(() -> "computed"); // lazy default
empty.orElseThrow(() -> new RuntimeException("No value!")); // throw

// Transform
present.map(String::toUpperCase);         // Optional["HELLO"]
present.filter(s -> s.length() > 3);      // Optional["Hello"]
present.filter(s -> s.length() > 10);     // Optional.empty

// ifPresent — execute only if has value
present.ifPresent(s -> System.out.println(s));  // prints "Hello"
empty.ifPresent(s -> System.out.println(s));    // nothing happens

// Real use case — chain nullable calls safely
// Instead of: if (user != null && user.getAddress() != null) ...
Optional.ofNullable(user)
    .map(u -> u.getAddress())
    .map(a -> a.getCity())
    .orElse("Unknown city");
⚠️ Don't use Optional as field type or method parameter — only as return type!
Optional is designed to be a return type for methods that might not return a value. Using it as a field wastes memory and breaks serialization.

New Date/Time API (java.time — Java 8)
The old java.util.Date and Calendar were notoriously broken (mutable, not thread-safe, confusing). Java 8 introduced java.time package with immutable, thread-safe, intuitive classes. 
import java.time.*;
import java.time.format.DateTimeFormatter;

// LocalDate — date only (no time, no timezone)
LocalDate today   = LocalDate.now();              // 2025-06-03
LocalDate specific = LocalDate.of(2024, 1, 15);   // 2024-01-15
LocalDate parsed  = LocalDate.parse("2024-06-15");

today.getYear();         // 2025
today.getMonthValue();   // 6
today.getDayOfMonth();   // 3
today.getDayOfWeek();    // TUESDAY
today.plusDays(7);       // 7 days from now (immutable — returns new)
today.minusMonths(1);   // 1 month ago
today.isAfter(specific); // true
today.isBefore(specific);// false

// LocalTime — time only
LocalTime now  = LocalTime.now();             // 14:30:00.123
LocalTime time = LocalTime.of(10, 30, 0);    // 10:30:00
time.getHour();   // 10
time.getMinute(); // 30

// LocalDateTime — date + time (no timezone)
LocalDateTime dt = LocalDateTime.now();
LocalDateTime dt2 = LocalDateTime.of(2024, 6, 15, 10, 30);

// ZonedDateTime — with timezone
ZonedDateTime zdt = ZonedDateTime.now(ZoneId.of("America/New_York"));

// Period — date-based duration (days, months, years)
Period p = Period.between(specific, today);
System.out.println(p.getYears() + " years, " + p.getMonths() + " months");

// Duration — time-based duration (hours, minutes, seconds)
Duration d = Duration.between(LocalTime.of(9, 0), LocalTime.of(17, 0));
System.out.println(d.toHours());  // 8

// Formatting
DateTimeFormatter fmt = DateTimeFormatter.ofPattern("dd/MM/yyyy");
System.out.println(today.format(fmt));              // 03/06/2025
LocalDate parsed2 = LocalDate.parse("15/06/2024", fmt);

Default & Static Interface Methods (Java 8)
interface Greeter {
    void greet(String name);  // abstract

    // Default method — has implementation
    default void greetLoud(String name) {
        greet(name.toUpperCase());
    }

    // Static method — utility
    static Greeter casual() {
        return name -> System.out.println("Hey, " + name + "!");
    }
}

Greeter formal = name -> System.out.println("Good morning, " + name);
formal.greet("Alice");       // Good morning, Alice
formal.greetLoud("Alice");   // Good morning, ALICE

Greeter c = Greeter.casual();
c.greet("Bob");              // Hey, Bob!
Real examples from Java 8 stdlib:
List.sort() — default method on List interface
Collection.stream() — default method on Collection
Map.forEach() — default method on Map
Comparator.comparing() — static method on Comparator

var keyword (Java 10)
// var — local variable type inference
var name   = "Alice";                  // inferred: String
var age    = 25;                       // inferred: int
var list   = new ArrayList<String>(); // inferred: ArrayList<String>
var map    = new HashMap<String, Integer>();

// Useful in loops
for (var entry : map.entrySet()) {
    System.out.println(entry.getKey() + " = " + entry.getValue());
}

// var is still strongly typed — just inferred
var x = 10;
// x = "hello"; ❌ compile error — x is int!

// var restrictions
// var x;          ❌ must initialize
// var x = null;   ❌ type unknown
// var[] arr;      ❌ no var arrays
// Only for local variables inside methods

Records (Java 16)
Records are immutable data classes. Java auto-generates constructor, getters, equals(), hashCode(), and toString(). Perfect for DTOs and value objects. 
// Before Records — verbose boilerplate
class PersonOld {
    private final String name;
    private final int    age;
    public PersonOld(String name, int age) { this.name = name; this.age = age; }
    public String getName() { return name; }
    public int    getAge()  { return age; }
    // + equals, hashCode, toString...
}

// With Records — all above in ONE LINE!
record Person(String name, int age) { }

Person p = new Person("Alice", 25);
System.out.println(p.name());       // Alice (accessor, not getName)
System.out.println(p.age());        // 25
System.out.println(p);              // Person[name=Alice, age=25]

Person p2 = new Person("Alice", 25);
System.out.println(p.equals(p2));   // true — content-based equals

// Records are immutable — no setters!
// p.name = "Bob"; ❌ fields are final

Top Interview Questions & Model Answers
Q1. What is a lambda expression? What is its purpose?
A lambda expression is a concise, anonymous function that implements a functional interface. Its purpose is to enable functional-style programming in Java, reducing boilerplate anonymous class code. Syntax: (params) -> expression. Lambdas can only be used where a functional interface (exactly one abstract method) is expected — like Runnable, Comparator, Predicate, etc.
Q2. What is the Stream API? What is the difference between intermediate and terminal operations?
Stream API provides functional-style operations on collections — filter, map, sort, reduce, collect. Intermediate operations (filter, map, sorted, distinct, limit) return a Stream and are lazy — they don't execute until a terminal operation is called. Terminal operations (collect, forEach, count, reduce, findFirst, anyMatch) trigger the pipeline execution and produce a result. Streams don't modify the source collection.
Q3. What is Optional? Why was it introduced?
Optional is a container object that may or may not hold a value. It was introduced to solve the NullPointerException problem — instead of returning null, methods return Optional, forcing callers to explicitly handle the empty case. Key methods: isPresent(), get(), orElse(), orElseGet(), orElseThrow(), map(), filter(), ifPresent(). Use it as a method return type — not as a field or parameter.
Q4. What is the difference between map() and flatMap() in Stream?
map() transforms each element into exactly one element — one-to-one transformation. flatMap() transforms each element into zero or more elements and flattens the result into a single stream — one-to-many. Example: map(s -> s.split("")) gives Stream<String[]>; flatMap(s -> Arrays.stream(s.split(""))) gives Stream<String> — the arrays are flattened.
Q5. What is a method reference? Give the 4 types.
A method reference is shorthand for a lambda that calls a single method. 4 types: (1) Static method: Integer::parseInt. (2) Instance method of specific object: System.out::println. (3) Instance method of any instance of a class: String::toUpperCase. (4) Constructor: ArrayList::new. All are equivalent to corresponding lambdas but more readable.
Q6. What is wrong with the old java.util.Date class?
java.util.Date is mutable — not thread-safe, can be changed unexpectedly. Month indexing is 0-based (January = 0) causing confusion. Date also contains time, making date-only comparisons tricky. Calendar class is complex and verbose. Java 8's java.time package (LocalDate, LocalTime, LocalDateTime) is immutable, thread-safe, clearly named, and uses proper month numbering (January = 1).
Q7. What are Records in Java? When would you use them?
Records (Java 16) are immutable data carrier classes. Declaring record Person(String name, int age) {} auto-generates: canonical constructor, accessors (name(), age()), equals(), hashCode(), and toString(). Fields are final — no setters. Use records for DTOs (Data Transfer Objects), value objects, and simple data containers where you want immutability and don't want to write boilerplate. They replace verbose POJO classes for pure data-holding purposes.
