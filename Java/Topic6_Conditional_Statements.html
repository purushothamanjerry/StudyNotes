1
if Statement
Definition: Executes a block of code only if the condition is true. 
// Syntax
if (condition) {
    // runs only if condition is true
}

// Example
int age = 20;
if (age >= 18) {
    System.out.println("You are an adult");
}

// Single line — braces optional (but NOT recommended!)
if (age >= 18)
    System.out.println("Adult");  // only this line is in if block
⚠️ Always use braces {} even for single statements!
Without braces, only the very next line belongs to the if block. Adding another line without braces looks like it's in the if block but isn't — a very common bug!
2
if-else Statement
// Syntax
if (condition) {
    // runs if condition is true
} else {
    // runs if condition is false
}

// Example
int num = 7;
if (num % 2 == 0) {
    System.out.println("Even");
} else {
    System.out.println("Odd");   // prints this
}

// Ternary is a shorthand for simple if-else
String result = (num % 2 == 0) ? "Even" : "Odd";
3
else-if Ladder

Used when you have multiple conditions to check. Only the first matching block runs — rest are skipped.

int score = 75;

if (score >= 90) {
    System.out.println("Grade: A");
} else if (score >= 70) {
    System.out.println("Grade: B");  // this runs
} else if (score >= 50) {
    System.out.println("Grade: C");
} else {
    System.out.println("Grade: F");
}
Key Rule: Once a condition is true, all remaining else if and else blocks are skipped entirely — even if they would also be true. 
// Tricky example — order matters!
int x = 100;
if (x > 10) {
    System.out.println("Greater than 10");  // this runs ONLY
} else if (x > 50) {
    System.out.println("Greater than 50");  // SKIPPED even though true!
} else if (x > 90) {
    System.out.println("Greater than 90");  // SKIPPED even though true!
}
4
Nested if

An if statement inside another if statement. Used when you need to check multiple related conditions.

int age = 20;
boolean hasID = true;

if (age >= 18) {
    if (hasID) {
        System.out.println("Entry allowed");   // prints this
    } else {
        System.out.println("Need ID card");
    }
} else {
    System.out.println("Too young");
}

// Same result using && — cleaner for simple cases
if (age >= 18 && hasID) {
    System.out.println("Entry allowed");
}
💡 Dangling else problem: When nested ifs don't have braces, the else always belongs to the nearest if. Always use braces to avoid confusion!
5
switch Statement

Used when you need to compare one variable against multiple fixed values. Cleaner than a long else-if ladder for exact value matching.

// Syntax
switch (variable) {
    case value1:
        // code
        break;
    case value2:
        // code
        break;
    default:
        // runs if no case matches (like else)
}

// Example
int day = 3;
switch (day) {
    case 1: System.out.println("Monday");    break;
    case 2: System.out.println("Tuesday");   break;
    case 3: System.out.println("Wednesday"); break;  // prints this
    case 4: System.out.println("Thursday");  break;
    default: System.out.println("Invalid day");
}

Fall-through — What happens without break?

⚠️ Without break, execution FALLS THROUGH to next cases! 
int x = 2;
switch (x) {
    case 1: System.out.println("One");
    case 2: System.out.println("Two");    // starts here
    case 3: System.out.println("Three");  // falls through!
    default: System.out.println("Default"); // falls through!
}
// Output: Two, Three, Default (all 3 print!)

// Intentional fall-through — multiple cases same action
switch (day) {
    case 1:
    case 2:
    case 3:
    case 4:
    case 5: System.out.println("Weekday"); break;
    case 6:
    case 7: System.out.println("Weekend"); break;
}

switch Expression (Java 14+)

Modern Java introduced a cleaner switch syntax using -> (arrow). No fall-through, no break needed, can return a value directly.

// Old switch statement
String result;
switch (day) {
    case 1: result = "Monday";    break;
    case 2: result = "Tuesday";   break;
    default: result = "Other";
}

// New switch expression — cleaner!
String result = switch (day) {
    case 1 -> "Monday";
    case 2 -> "Tuesday";
    case 3 -> "Wednesday";
    default -> "Other";
};

// Multiple values per case
String type = switch (day) {
    case 1, 2, 3, 4, 5 -> "Weekday";
    case 6, 7          -> "Weekend";
    default             -> "Invalid";
};
Old switch vs New switch expression:
Old: uses : and needs break, has fall-through, is a statement
New: uses ->, no break needed, no fall-through, can be an expression (returns value)

if-else vs switch — When to Use Which
Featureif-elseswitch
Condition typeAny boolean expression (ranges, complex)Exact value matching only
Data typesAny typeint, char, String, enum only
ReadabilityGood for 2-3 conditionsBetter for 4+ exact values
Range checks✅ x > 10 && x < 20❌ Not possible
Null safe✅ Can check null❌ Throws NPE on null
Fall-throughNot applicableYes (without break)

Use if-else when:

  • Checking ranges: age >= 18
  • Complex conditions: a > b && c != 0
  • Checking null
  • Only 2-3 conditions

Use switch when:

  • Checking exact values: day == 1
  • Many fixed cases (menu options)
  • String or enum matching
  • Cleaner code for 4+ cases

Top Interview Questions & Model Answers
Q1. What is fall-through in switch? How do you prevent it?
Fall-through means when a case matches and executes, execution continues into the next case automatically without stopping — even if the next case doesn't match. It happens when break is missing. To prevent it, add break at the end of every case. In Java 14+, using the arrow syntax -> in switch expressions prevents fall-through automatically.
Q2. What is the output? 
int x = 2;
switch(x) {
  case 1: System.out.println("One");
  case 2: System.out.println("Two");
  case 3: System.out.println("Three");
  default: System.out.println("Default");
}
Two, Three, Default — because there are no break statements. Execution starts at case 2 and falls through all remaining cases including default.
Q3. Can you use String in a switch statement?
Yes, from Java 7 onwards. Before Java 7, switch only supported primitive types and their wrappers. Note: never pass null to a switch with String — it throws NullPointerException. switch uses .equals() internally to compare strings.
Q4. What is the difference between if-else and switch?
if-else can handle any boolean expression including ranges and complex conditions, works with any data type, and is better for 2-3 conditions. switch only handles exact value matching, supports int, char, String, and enum types, and is cleaner for 4+ exact value cases. switch cannot check null (NPE) while if-else can.
Q5. What is the output of this code? 
int x = 100;
if (x > 10) {
    System.out.println("A");
} else if (x > 50) {
    System.out.println("B");
} else if (x > 90) {
    System.out.println("C");
}
A only — even though x is also greater than 50 and 90, the first matching condition x > 10 runs and all remaining else-if blocks are skipped. Order matters in else-if ladders — always put more specific conditions first!
Q6. What is a switch expression in Java 14+? How is it different from switch statement?
Switch expression uses arrow syntax ->, returns a value directly, requires no break statements, and has no fall-through. Switch statement uses colon syntax :, needs explicit break to prevent fall-through, and cannot directly return a value. Switch expressions make code cleaner and less error-prone.
Q7. What data types are NOT supported in switch?
long, float, double, and boolean are not supported in switch statements. Switch supports int, byte, short, char, their wrapper classes, String (Java 7+), and enum.
