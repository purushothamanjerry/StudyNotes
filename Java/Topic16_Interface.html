What is an Interface?
Definition: An interface is a pure contract — it defines what a class must do, but not how. It is declared with the interface keyword. Any class that implements an interface must provide implementations for all its abstract methods. An interface cannot be instantiated.

Real World Analogy

A job contract — it lists what you must do (responsibilities). It doesn't say HOW you do it. Any employee who signs the contract must fulfill those responsibilities in their own way.

Key Ideas

  • Interface = contract/capability
  • Defines WHAT, not HOW
  • Enables multiple inheritance
  • Achieves loose coupling
  • Enables polymorphism across unrelated classes

Syntax & Basic Example
// Interface declaration
interface Flyable {
    // Constants — implicitly public static final
    int MAX_HEIGHT = 10000;  // same as: public static final int

    // Abstract methods — implicitly public abstract
    void fly();    // same as: public abstract void fly();
    void land();
}

interface Swimmable {
    void swim();
}

// Class implements ONE interface
class Bird implements Flyable {
    @Override
    public void fly()  { System.out.println("Bird flying"); }
    @Override
    public void land() { System.out.println("Bird landing"); }
}

// Class implements MULTIPLE interfaces — multiple inheritance!
class Duck implements Flyable, Swimmable {
    @Override
    public void fly()  { System.out.println("Duck flying"); }
    @Override
    public void land() { System.out.println("Duck landing"); }
    @Override
    public void swim() { System.out.println("Duck swimming"); }
}

// Interface reference — polymorphism
Flyable f = new Duck();
f.fly();    // Duck flying
System.out.println(Flyable.MAX_HEIGHT);  // 10000

Multiple Interface Implementation
interface Printable   { void print(); }
interface Saveable    { void save();  }
interface Shareable   { void share(); }

// Implements 3 interfaces — multiple inheritance of type!
class Document implements Printable, Saveable, Shareable {
    @Override public void print() { System.out.println("Printing..."); }
    @Override public void save()  { System.out.println("Saving...");   }
    @Override public void share() { System.out.println("Sharing...");  }
}

// Class + interface — extends ONE class, implements MANY
class SpreadSheet extends Document implements Printable, Saveable {
    // inherits Document's implementations
}

// Use interface as reference — polymorphism across unrelated classes
Printable[] printables = { new Document(), new SpreadSheet() };
for (Printable p : printables) p.print();

+ default methods

+ private methods

Why did Java 8 add default methods?
To add new methods to existing interfaces (like Collection.stream(), List.sort()) WITHOUT breaking all existing implementations. Before default methods, adding a method to an interface would break every class that implements it.

Default Methods (Java 8)
Default method — a method in an interface with a body (implementation). Implementing classes can use it as-is or override it. Introduced for backward compatibility. 
interface Vehicle {
    void start();   // abstract — must implement

    // Default method — has body, optional to override
    default void stop() {
        System.out.println("Vehicle stopped");
    }

    default void fuelStatus() {
        System.out.println("Checking fuel...");
    }
}

class Car implements Vehicle {
    @Override
    public void start() { System.out.println("Car started"); }
    // stop() and fuelStatus() — inherited from interface, no need to override
}

class Truck implements Vehicle {
    @Override
    public void start() { System.out.println("Truck started"); }

    @Override
    public void stop() {  // Override default
        System.out.println("Truck applying air brakes...");
    }
}

Car c = new Car();
c.start();      // Car started
c.stop();       // Vehicle stopped (default)

Truck t = new Truck();
t.stop();       // Truck applying air brakes (overridden)

Diamond Problem with Default Methods

interface A { default void greet() { System.out.println("Hello from A"); } }
interface B { default void greet() { System.out.println("Hello from B"); } }

// Conflict — must override to resolve!
class C implements A, B {
    @Override
    public void greet() {
        A.super.greet();  // explicitly choose A's version
        // or B.super.greet();
        // or write your own implementation
    }
}

Static Methods (Java 8)
Static methods in interfaces — utility/helper methods that belong to the interface itself. Called via the interface name. NOT inherited by implementing classes. 
interface MathUtils {
    // Static method — utility function
    static int square(int n) { return n * n; }
    static boolean isEven(int n) { return n % 2 == 0; }
}

// Call via INTERFACE NAME only
int s = MathUtils.square(5);        // ✅ 25
boolean e = MathUtils.isEven(4);   // ✅ true

class MyClass implements MathUtils { }
MyClass obj = new MyClass();
// obj.square(5);       ❌ NOT accessible on instance
// MyClass.square(5);   ❌ NOT inherited by implementing class
⚠️ Interface static methods are NOT inherited!
Unlike class static methods, interface static methods cannot be called on implementing class or its objects. Only via the interface name directly.

Private Methods (Java 9)

Private methods in interfaces are helper methods used internally by default or static methods. Not visible to implementing classes.

interface Logger {
    // Default methods use private helper
    default void logInfo(String msg)  { log("INFO",  msg); }
    default void logError(String msg) { log("ERROR", msg); }
    default void logWarn(String msg)  { log("WARN",  msg); }

    // Private helper — avoids code duplication
    private void log(String level, String msg) {
        System.out.println("[" + level + "] " + msg);
    }

    // Private static — for static method helpers
    private static String timestamp() {
        return "2025";
    }
}

class AppLogger implements Logger {
    // Inherits logInfo, logError, logWarn
    // private log() is NOT accessible here
}

AppLogger logger = new AppLogger();
logger.logInfo("App started");   // [INFO] App started
logger.logError("Error!");      // [ERROR] Error!

Functional Interface & Lambda (Java 8)
Functional Interface = an interface with exactly ONE abstract method. Annotated with @FunctionalInterface (optional but recommended). Can be implemented using a lambda expression. 
// Custom Functional Interface
@FunctionalInterface
interface Calculator {
    int calculate(int a, int b);
    // only ONE abstract method allowed
    // can have default/static methods — they don't count
}

// Old way — anonymous class
Calculator add = new Calculator() {
    public int calculate(int a, int b) { return a + b; }
};

// New way — lambda expression (Java 8)
Calculator add = (a, b) -> a + b;
Calculator mul = (a, b) -> a * b;
Calculator max = (a, b) -> a > b ? a : b;

System.out.println(add.calculate(3, 4));  // 7
System.out.println(mul.calculate(3, 4));  // 12

Marker Interface
Marker interface = an interface with NO methods. It simply "marks" or "tags" a class to signal some capability or property to the JVM or framework. 
// Built-in marker interfaces
interface Serializable  { }  // marks class for serialization
interface Cloneable     { }  // marks class for cloning
interface Remote        { }  // marks class for RMI

// Using Serializable — JVM checks this marker
import java.io.Serializable;

class Student implements Serializable {
    String name;
    int    age;
    // JVM allows this object to be serialized to file/network
}

// Without Serializable:
// ObjectOutputStream.writeObject(student) → ❌ NotSerializableException!

// Custom marker interface
interface Auditable { }  // mark entities that should be audited

class Order implements Auditable { }

// Check with instanceof
if (obj instanceof Auditable) {
    System.out.println("This object needs auditing");
}

Interface Extending Interface
interface Animal    { void breathe(); }
interface Pet       { void beFriendly(); }

// Interface extends MULTIPLE interfaces — no Diamond Problem!
interface DomesticAnimal extends Animal, Pet {
    void livesWithHumans();
}

// Implementing class must implement ALL methods
class Dog implements DomesticAnimal {
    @Override public void breathe()          { System.out.println("Breathing"); }
    @Override public void beFriendly()       { System.out.println("Wagging tail"); }
    @Override public void livesWithHumans()  { System.out.println("Lives at home"); }
}
💡 Key distinction:
Interface extends interface → use extends keyword (multiple allowed)
Class implements interface → use implements keyword (multiple allowed)
Class extends class → use extends keyword (only ONE allowed)

Top Interview Questions & Model Answers
Q1. What is an interface? How is it different from a class?
An interface is a pure contract that defines what a class must do — it cannot be instantiated and has no constructor or instance variables. Before Java 8, it only had abstract methods and constants. Unlike a class, a class can implement multiple interfaces (multiple inheritance of type), while it can only extend one class. Interfaces define capability (Flyable, Runnable), while classes define entities (Dog, Car).
Q2. What are default methods? Why were they added in Java 8?
Default methods are methods in interfaces that have a body — implementing classes can use them without overriding. They were added to allow adding new methods to existing interfaces without breaking all existing implementations (backward compatibility). For example, the stream() method was added to Collection interface in Java 8 as a default method — all existing classes like ArrayList didn't need to change.
Q3. What is a functional interface? Give examples.
A functional interface has exactly ONE abstract method. It can be implemented using a lambda expression. The @FunctionalInterface annotation ensures this contract. Built-in examples: Runnable (run()), Comparable (compareTo()), Predicate<T> (test()), Function<T,R> (apply()), Consumer<T> (accept()), Supplier<T> (get()).
Q4. What is a marker interface? Give examples.
A marker interface has no methods — it simply tags a class to signal a capability to the JVM or framework. Examples: Serializable (marks for serialization), Cloneable (marks for cloning), Remote (marks for RMI). Modern Java uses annotations (like @Override, @Deprecated) for the same purpose, which is more flexible.
Q5. What happens when two interfaces have the same default method and a class implements both?
This is the Diamond Problem with default methods. The class MUST override the conflicting method to resolve the ambiguity — otherwise the compiler throws an error. Inside the override, you can choose which interface's version to call using InterfaceName.super.methodName() syntax, or write your own implementation entirely.
Q6. Can interface have static methods? Are they inherited?
Yes, since Java 8. But unlike class static methods, interface static methods are NOT inherited by implementing classes. They can only be called via the interface name directly (e.g., MathUtils.square(5)). Calling them on an implementing class or object causes a compile error. This is a key difference from abstract class static methods.
Q7. What is the difference between interface and abstract class? When to use which?
Abstract class: can have constructors, instance variables, any access modifiers, only extends one class. Use when classes are closely related and share state. Interface: no constructor, only constants, methods public by default, can implement multiple. Use when unrelated classes need a common capability, or when multiple inheritance is needed. Modern rule: prefer interfaces unless you need constructors or instance variables.
