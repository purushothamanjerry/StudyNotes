Collections Framework Overview
The Java Collections Framework (JCF) is a unified architecture for storing and manipulating groups of objects. It provides ready-made data structures (List, Set, Map, Queue) and algorithms (sort, search, shuffle) — no need to build from scratch.
  • java.util package — where all collections live
  • Based on interfaces — code to interface, not implementation
  • All use Generics for type safety
  • Replaces arrays for most use cases — dynamic size, rich API

Collection (stores individual objects)

  • List — ordered, allows duplicates
  • Set — no duplicates
  • Queue — FIFO ordering
  • Deque — double-ended queue

Map (stores key-value pairs)

  • HashMap — no order, fast
  • LinkedHashMap — insertion order
  • TreeMap — sorted by key
  • Map does NOT extend Collection!

List — ArrayList & LinkedList
List = ordered collection that allows duplicates. Elements have an index (0-based). Most used collection type.

ArrayList

import java.util.*;

List<String> list = new ArrayList<>();

// Add
list.add("Alice");
list.add("Bob");
list.add("Charlie");
list.add(1, "Dave");   // insert at index 1

// Access
System.out.println(list.get(0));    // Alice
System.out.println(list.size());     // 4
System.out.println(list.contains("Bob")); // true
System.out.println(list.indexOf("Bob"));  // index of Bob

// Update
list.set(0, "Alex");  // replace index 0

// Remove
list.remove("Bob");    // remove by value
list.remove(0);        // remove by index

// Iterate
for (String s : list) System.out.println(s);

// Sort
Collections.sort(list);                         // natural order
list.sort(Comparator.reverseOrder());            // reverse
list.sort((a, b) -> a.length() - b.length());    // by length

// Convert array → List and List → array
String[] arr  = {"x", "y"};
List<String> fromArr = new ArrayList<>(Arrays.asList(arr));
String[] toArr = list.toArray(new String[0]);

LinkedList

LinkedList<String> ll = new LinkedList<>();
ll.addFirst("A");    // add to front
ll.addLast("B");     // add to end
ll.add("C");
System.out.println(ll.getFirst());  // A
System.out.println(ll.getLast());   // C
ll.removeFirst();
ll.removeLast();

Set — HashSet, LinkedHashSet, TreeSet
Set = collection that does NOT allow duplicates. Adding a duplicate is silently ignored (no error). No index-based access. 
// HashSet — fastest, NO order guaranteed
Set<String> hashSet = new HashSet<>();
hashSet.add("Banana");
hashSet.add("Apple");
hashSet.add("Cherry");
hashSet.add("Apple");   // duplicate — silently ignored!
System.out.println(hashSet.size());        // 3
System.out.println(hashSet.contains("Apple")); // true

// LinkedHashSet — maintains INSERTION ORDER
Set<String> linkedSet = new LinkedHashSet<>();
linkedSet.add("Banana");
linkedSet.add("Apple");
linkedSet.add("Cherry");
System.out.println(linkedSet);  // [Banana, Apple, Cherry] — insertion order

// TreeSet — SORTED (natural order or custom Comparator)
Set<String> treeSet = new TreeSet<>();
treeSet.add("Banana");
treeSet.add("Apple");
treeSet.add("Cherry");
System.out.println(treeSet);    // [Apple, Banana, Cherry] — sorted!

// Set operations
Set<Integer> setA = new HashSet<>(Arrays.asList(1, 2, 3, 4));
Set<Integer> setB = new HashSet<>(Arrays.asList(3, 4, 5, 6));
setA.retainAll(setB);  // intersection: {3, 4}
setA.addAll(setB);     // union
setA.removeAll(setB);  // difference

Map — HashMap, LinkedHashMap, TreeMap
Map stores key-value pairs. Keys are unique — adding same key overwrites value. Values can duplicate. Map does NOT extend Collection. 
// HashMap — most used, NO order
Map<String, Integer> map = new HashMap<>();

// Put (add/update)
map.put("Alice", 25);
map.put("Bob",   30);
map.put("Alice", 26);  // overwrites Alice's value!

// Get
System.out.println(map.get("Alice"));         // 26
System.out.println(map.get("Unknown"));        // null
System.out.println(map.getOrDefault("X", 0)); // 0 (safe)

// Check
map.containsKey("Alice");    // true
map.containsValue(30);        // true
map.size();                     // 2

// Remove
map.remove("Bob");

// Iterate — 3 ways
// 1. entrySet — most efficient
for (Map.Entry<String, Integer> entry : map.entrySet()) {
    System.out.println(entry.getKey() + " → " + entry.getValue());
}
// 2. keySet
for (String key : map.keySet()) {
    System.out.println(key + " → " + map.get(key));
}
// 3. forEach (Java 8)
map.forEach((k, v) -> System.out.println(k + " → " + v));

// putIfAbsent — only add if key doesn't exist
map.putIfAbsent("Alice", 99);  // won't change Alice's value
map.putIfAbsent("Carol", 22);  // adds Carol

// merge — useful for counting
String[] words = {"apple", "banana", "apple", "cherry", "apple"};
Map<String, Integer> freq = new HashMap<>();
for (String w : words) {
    freq.merge(w, 1, Integer::sum);
}
System.out.println(freq);  // {apple=3, banana=1, cherry=1}

LinkedHashMap & TreeMap

// LinkedHashMap — maintains INSERTION ORDER
Map<String, Integer> linked = new LinkedHashMap<>();
linked.put("C", 3); linked.put("A", 1); linked.put("B", 2);
System.out.println(linked);  // {C=3, A=1, B=2} — insertion order

// TreeMap — SORTED by key (natural or custom)
Map<String, Integer> tree = new TreeMap<>();
tree.put("C", 3); tree.put("A", 1); tree.put("B", 2);
System.out.println(tree);    // {A=1, B=2, C=3} — sorted by key!

// TreeMap extra methods
TreeMap<Integer, String> tm = new TreeMap<>();
tm.put(10, "ten"); tm.put(20, "twenty"); tm.put(30, "thirty");
tm.firstKey();     // 10
tm.lastKey();      // 30
tm.headMap(20);    // {10=ten} — keys less than 20
tm.tailMap(20);    // {20=twenty, 30=thirty}

Queue & Deque
// Queue — FIFO (First In, First Out)
Queue<String> queue = new LinkedList<>();
queue.offer("Alice");   // add to tail (safe — returns false if full)
queue.offer("Bob");
queue.offer("Charlie");

System.out.println(queue.peek());   // Alice — view front, don't remove
System.out.println(queue.poll());   // Alice — view AND remove
System.out.println(queue.size());   // 2

// PriorityQueue — elements ordered by priority (min-heap by default)
PriorityQueue<Integer> pq = new PriorityQueue<>();
pq.add(5); pq.add(1); pq.add(3);
System.out.println(pq.poll());  // 1 (smallest first!)
System.out.println(pq.poll());  // 3

// Max heap
PriorityQueue<Integer> maxPQ = new PriorityQueue<>(Collections.reverseOrder());
maxPQ.add(5); maxPQ.add(1); maxPQ.add(3);
System.out.println(maxPQ.poll()); // 5 (largest first!)

// Deque (Double-Ended Queue) — add/remove from both ends
Deque<String> deque = new ArrayDeque<>();
deque.addFirst("B");
deque.addFirst("A");
deque.addLast("C");
System.out.println(deque);        // [A, B, C]
deque.pollFirst();                // remove A
deque.pollLast();                 // remove C

// ArrayDeque as Stack (faster than Stack class)
Deque<Integer> stack = new ArrayDeque<>();
stack.push(1);  // LIFO push
stack.push(2);
stack.push(3);
System.out.println(stack.pop());  // 3 (LIFO)

Collections Utility Class
List<Integer> nums = new ArrayList<>(Arrays.asList(3, 1, 4, 1, 5, 9));

Collections.sort(nums);                          // [1, 1, 3, 4, 5, 9]
Collections.sort(nums, Comparator.reverseOrder()); // [9, 5, 4, 3, 1, 1]
Collections.reverse(nums);                       // reverses the list
Collections.shuffle(nums);                       // random shuffle
Collections.min(nums);                           // smallest element
Collections.max(nums);                           // largest element
Collections.frequency(nums, 1);                 // count of value 1 → 2
Collections.fill(nums, 0);                      // set all to 0
Collections.binarySearch(nums, 4);              // find index (must be sorted!)
Collections.swap(nums, 0, 1);                   // swap elements at index 0 and 1

// Unmodifiable — read-only wrapper
List<String> immutable = Collections.unmodifiableList(list);
// immutable.add("x"); ❌ UnsupportedOperationException

// Synchronized — thread-safe wrapper
List<String> synced = Collections.synchronizedList(new ArrayList<>());

// Singleton — immutable single-element collection
List<String> single = Collections.singletonList("only");

// Empty collections
List<String>  emptyList = Collections.emptyList();
Set<String>   emptySet  = Collections.emptySet();
Map<String,Integer> emptyMap = Collections.emptyMap();

Comparable vs Comparator
// Comparable — natural ordering (built into the class)
public class Student implements Comparable<Student> {
    String name;
    int    age;

    @Override
    public int compareTo(Student other) {
        return this.age - other.age;  // sort by age ascending
    }
}
List<Student> students = ...;
Collections.sort(students);  // uses compareTo

// Comparator — external/custom ordering
Comparator<Student> byName = (a, b) -> a.name.compareTo(b.name);
Comparator<Student> byAge  = Comparator.comparingInt(s -> s.age);
Comparator<Student> byAgeDesc = byAge.reversed();
Comparator<Student> byNameThenAge = byName.thenComparingInt(s -> s.age);

students.sort(byName);          // sort by name
students.sort(byAge.reversed()); // sort by age desc

Iterator & Safe Removal
List<String> list = new ArrayList<>(Arrays.asList("a", "b", "c", "d"));

// Iterator — safe way to remove while iterating
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String s = it.next();
    if (s.equals("b")) {
        it.remove();  // ✅ safe removal via iterator
    }
}
System.out.println(list);  // [a, c, d]

// BAD — modifying list during for-each → ConcurrentModificationException!
for (String s : list) {
    // list.remove(s); ❌ ConcurrentModificationException!
}

// Java 8 — removeIf (cleanest)
list.removeIf(s -> s.equals("c"));  // ✅ safe and clean

Default choice → ArrayList for list, HashMap for map, HashSet for set
Need order → LinkedHashMap/LinkedHashSet
Need sorted → TreeMap/TreeSet

Top Interview Questions & Model Answers
Q1. What is the difference between ArrayList and LinkedList?
ArrayList uses a dynamic array internally — O(1) for random access (get by index), but O(n) for insert/delete in the middle (elements must shift). LinkedList uses doubly-linked nodes — O(1) for insert/delete at any position once you have the node, but O(n) for random access. Use ArrayList for frequent reads; LinkedList for frequent middle insertions/deletions.
Q2. What is the difference between HashMap and TreeMap?
HashMap stores key-value pairs with no ordering — O(1) for get/put using hashing. TreeMap stores entries sorted by key using a Red-Black tree — O(log n) for get/put. HashMap allows one null key; TreeMap does not allow null keys (comparison would throw NPE). Use HashMap for fast lookups; TreeMap when you need keys in sorted order.
Q3. What is the difference between HashSet and TreeSet?
HashSet stores elements with no ordering — O(1) for add/contains/remove using hashing. TreeSet stores elements in sorted (natural or custom) order using a Red-Black tree — O(log n) for operations. HashSet allows one null; TreeSet does not. Use HashSet for fast membership testing; TreeSet when you need sorted unique elements.
Q4. What is the difference between Comparable and Comparator?
Comparable is implemented by the class itself (java.lang) with the compareTo() method — defines the natural/default sort order. Comparator is an external comparison strategy (java.util) with the compare() method — defines custom sort orders. Use Comparable for one default sort; use Comparator when you need multiple different sort orders or can't modify the class.
Q5. How does HashMap work internally?
HashMap uses an array of buckets. When you put(key, value): (1) hashCode() of key is calculated, (2) hash is mapped to a bucket index, (3) key-value is stored in that bucket as a linked list node (or tree node if chain > 8). On get(key): same hash calculation, find bucket, compare keys using equals(). If two keys have the same hash (collision), they go in the same bucket and are chained. Requires that equal objects have equal hash codes — why you must override both equals() and hashCode() together.
Q6. What is ConcurrentModificationException? How do you avoid it?
ConcurrentModificationException is thrown when you modify a collection while iterating over it with a for-each loop. Avoid it by: (1) Using Iterator.remove() instead of collection.remove() during iteration. (2) Using list.removeIf(predicate) (Java 8). (3) Collecting elements to remove in a separate list, then call removeAll after iteration. (4) Using concurrent collections like CopyOnWriteArrayList.
Q7. What is the difference between fail-fast and fail-safe iterators?
Fail-fast iterators (ArrayList, HashMap) throw ConcurrentModificationException if the collection is modified during iteration — they detect structural changes via a modCount counter. Fail-safe iterators (CopyOnWriteArrayList, ConcurrentHashMap) work on a copy or snapshot of the collection — no exception, but may not reflect recent changes. Use fail-safe collections in multi-threaded environments.
