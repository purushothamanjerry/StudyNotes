What is JDBC?
JDBC (Java Database Connectivity) is a Java API that allows Java programs to connect to and interact with relational databases (MySQL, PostgreSQL, Oracle, SQLite, etc.). It is in the java.sql package and provides a standard interface regardless of the database vendor.

JDBC Architecture

Java App → JDBC API → JDBC Driver → Database

JDBC Driver is database-specific (MySQL driver, PostgreSQL driver)

Your code stays the same — just change the driver!

Key JDBC Interfaces (java.sql)

  • Connection — database session
  • Statement — execute SQL
  • PreparedStatement — precompiled SQL
  • ResultSet — query results
  • CallableStatement — stored procedures

Step 1 — Load Driver & Connect
import java.sql.*;

// MySQL connection URL format:
// jdbc:mysql://host:port/database?params
String url      = "jdbc:mysql://localhost:3306/mydb?useSSL=false";
String username = "root";
String password = "password";

// JDBC 4.0+ — driver auto-loaded from classpath (no Class.forName needed)
// Older code: Class.forName("com.mysql.cj.jdbc.Driver");

try (Connection conn = DriverManager.getConnection(url, username, password)) {
    System.out.println("Connected! DB: " + conn.getCatalog());
    System.out.println("AutoCommit: " + conn.getAutoCommit());  // true by default
} catch (SQLException e) {
    System.out.println("Connection failed: " + e.getMessage());
}

// Common JDBC URLs
// MySQL:      jdbc:mysql://localhost:3306/dbname
// PostgreSQL: jdbc:postgresql://localhost:5432/dbname
// SQLite:     jdbc:sqlite:./mydb.db
// H2:         jdbc:h2:mem:testdb  (in-memory, great for testing)
// Oracle:     jdbc:oracle:thin:@localhost:1521:orcl

Step 2 — Statement
Statement — executes static SQL. Simple but vulnerable to SQL injection. Use for DDL (CREATE, DROP) or when no user input is involved. 
try (Connection  conn = DriverManager.getConnection(url, user, pass);
     Statement  stmt = conn.createStatement()) {

    // executeQuery — for SELECT, returns ResultSet
    ResultSet rs = stmt.executeQuery("SELECT * FROM students");

    // executeUpdate — for INSERT, UPDATE, DELETE, DDL
    // returns number of rows affected
    int rows = stmt.executeUpdate("INSERT INTO students (name, age) VALUES ('Alice', 20)");
    System.out.println(rows + " row(s) inserted");

    // execute — for any SQL, returns boolean
    // true = ResultSet, false = update count
    boolean hasResult = stmt.execute("CREATE TABLE test (id INT)");

    // Batch execution — multiple statements at once
    stmt.addBatch("INSERT INTO students VALUES (1, 'Bob', 21)");
    stmt.addBatch("INSERT INTO students VALUES (2, 'Carol', 22)");
    int[] results = stmt.executeBatch();
}

// ⚠️ SQL Injection risk with Statement!
String name = "' OR '1'='1";  // malicious input
stmt.executeQuery("SELECT * FROM users WHERE name='" + name + "'");
// Becomes: SELECT * FROM users WHERE name='' OR '1'='1'
// Returns ALL users! Security breach!

PreparedStatement (Preferred)
PreparedStatement — precompiled SQL with placeholders (?). Safer (prevents SQL injection) and faster (compiled once, executed many times). Always use over Statement when user input is involved. 
String sql = "INSERT INTO students (name, age, email) VALUES (?, ?, ?)";

try (Connection        conn = DriverManager.getConnection(url, user, pass);
     PreparedStatement ps   = conn.prepareStatement(sql)) {

    // Set parameters by index (1-based!)
    ps.setString(1, "Alice");
    ps.setInt(2, 20);
    ps.setString(3, "alice@email.com");
    int rows = ps.executeUpdate();

    // Reuse with different values
    ps.setString(1, "Bob");
    ps.setInt(2, 22);
    ps.setString(3, "bob@email.com");
    ps.executeUpdate();
}

// SELECT with PreparedStatement
String query = "SELECT * FROM students WHERE age > ? AND name LIKE ?";
try (Connection        conn = DriverManager.getConnection(url, user, pass);
     PreparedStatement ps   = conn.prepareStatement(query)) {

    ps.setInt(1, 18);
    ps.setString(2, "%A%");   // names containing 'A'
    ResultSet rs = ps.executeQuery();
    while (rs.next()) {
        System.out.println(rs.getString("name") + " - " + rs.getInt("age"));
    }
}

// setXxx methods
// ps.setString(i, val)    — varchar/text
// ps.setInt(i, val)       — integer
// ps.setDouble(i, val)    — double
// ps.setBoolean(i, val)   — boolean
// ps.setDate(i, sqlDate)  — date
// ps.setNull(i, Types.INTEGER) — null value
// ps.setObject(i, val)    — any type
💡 Why PreparedStatement prevents SQL Injection:
The SQL structure is fixed at preparation time. User input is treated as a literal value, never as SQL code. 'OR '1'='1 is just a string, not executed as SQL.

Step 4 — ResultSet
ResultSet is a table of data representing the result of a SELECT query. The cursor starts BEFORE the first row — call next() to move to each row. 
ResultSet rs = stmt.executeQuery("SELECT id, name, age, email FROM students");

// Iterate rows
while (rs.next()) {   // moves cursor forward, returns false at end

    // Read by column NAME (preferred — readable)
    int    id    = rs.getInt("id");
    String name  = rs.getString("name");
    int    age   = rs.getInt("age");
    String email = rs.getString("email");

    // Read by column INDEX (1-based)
    int    id2   = rs.getInt(1);
    String name2 = rs.getString(2);

    System.out.printf("ID: %d | Name: %s | Age: %d%n", id, name, age);
}

// Metadata — info about columns
ResultSetMetaData meta = rs.getMetaData();
int columnCount = meta.getColumnCount();  // number of columns
for (int i = 1; i <= columnCount; i++) {
    System.out.println(meta.getColumnName(i) + " - " + meta.getColumnTypeName(i));
}

// Check null values
String phone = rs.getString("phone");
if (rs.wasNull()) {
    System.out.println("Phone is NULL");
}

Full CRUD Example
import java.sql.*;

public class StudentDAO {

    private static final String URL  = "jdbc:mysql://localhost:3306/school";
    private static final String USER = "root";
    private static final String PASS = "password";

    // CREATE
    public void insertStudent(String name, int age) {
        String sql = "INSERT INTO students (name, age) VALUES (?, ?)";
        try (Connection conn = DriverManager.getConnection(URL, USER, PASS);
             PreparedStatement ps = conn.prepareStatement(sql)) {
            ps.setString(1, name);
            ps.setInt(2, age);
            int rows = ps.executeUpdate();
            System.out.println(rows + " student inserted");
        } catch (SQLException e) { e.printStackTrace(); }
    }

    // READ ALL
    public void getAllStudents() {
        String sql = "SELECT * FROM students";
        try (Connection conn = DriverManager.getConnection(URL, USER, PASS);
             Statement  stmt = conn.createStatement();
             ResultSet  rs   = stmt.executeQuery(sql)) {
            while (rs.next()) {
                System.out.printf("ID: %d | Name: %-15s | Age: %d%n",
                    rs.getInt("id"), rs.getString("name"), rs.getInt("age"));
            }
        } catch (SQLException e) { e.printStackTrace(); }
    }

    // UPDATE
    public void updateAge(int id, int newAge) {
        String sql = "UPDATE students SET age = ? WHERE id = ?";
        try (Connection conn = DriverManager.getConnection(URL, USER, PASS);
             PreparedStatement ps = conn.prepareStatement(sql)) {
            ps.setInt(1, newAge);
            ps.setInt(2, id);
            System.out.println(ps.executeUpdate() + " row(s) updated");
        } catch (SQLException e) { e.printStackTrace(); }
    }

    // DELETE
    public void deleteStudent(int id) {
        String sql = "DELETE FROM students WHERE id = ?";
        try (Connection conn = DriverManager.getConnection(URL, USER, PASS);
             PreparedStatement ps = conn.prepareStatement(sql)) {
            ps.setInt(1, id);
            System.out.println(ps.executeUpdate() + " row(s) deleted");
        } catch (SQLException e) { e.printStackTrace(); }
    }
}

Transactions
Transaction = a group of SQL operations that must ALL succeed or ALL fail together. Uses ACID properties: Atomicity, Consistency, Isolation, Durability. 
// By default, autoCommit = true (each statement commits immediately)
// For transactions, disable autoCommit

try (Connection conn = DriverManager.getConnection(url, user, pass)) {

    conn.setAutoCommit(false);  // START TRANSACTION

    try {
        // Multiple operations in one transaction
        try (PreparedStatement ps1 = conn.prepareStatement(
                "UPDATE accounts SET balance = balance - ? WHERE id = ?")) {
            ps1.setDouble(1, 500.0);
            ps1.setInt(2, 1);         // debit from account 1
            ps1.executeUpdate();
        }

        try (PreparedStatement ps2 = conn.prepareStatement(
                "UPDATE accounts SET balance = balance + ? WHERE id = ?")) {
            ps2.setDouble(1, 500.0);
            ps2.setInt(2, 2);         // credit to account 2
            ps2.executeUpdate();
        }

        conn.commit();   // ✅ BOTH succeed — commit
        System.out.println("Transfer successful!");

    } catch (SQLException e) {
        conn.rollback(); // ❌ ANY failure — rollback both
        System.out.println("Transfer failed, rolled back: " + e.getMessage());
    }

} catch (SQLException e) {
    e.printStackTrace();
}

Savepoints

// Savepoints — partial rollback
conn.setAutoCommit(false);
Savepoint sp1 = conn.setSavepoint("afterInsert");
try {
    // more operations...
    conn.commit();
} catch (SQLException e) {
    conn.rollback(sp1);  // rollback only to savepoint, not all
    conn.commit();       // commit everything up to savepoint
}

Connection Pooling
Creating a new database connection for every request is expensive (time + resources). Connection pooling maintains a pool of pre-created connections that are reused. Essential for production applications.

HikariCP — Most Popular Connection Pool

// Add HikariCP dependency (Maven)
// <dependency><groupId>com.zaxxer</groupId>
//   <artifactId>HikariCP</artifactId><version>5.0.1</version></dependency>

import com.zaxxer.hikari.*;

HikariConfig config = new HikariConfig();
config.setJdbcUrl("jdbc:mysql://localhost:3306/school");
config.setUsername("root");
config.setPassword("password");
config.setMaximumPoolSize(10);     // max 10 connections
config.setMinimumIdle(2);          // keep 2 idle connections
config.setConnectionTimeout(30000); // 30 second timeout

HikariDataSource dataSource = new HikariDataSource(config);

// Use just like DriverManager
try (Connection conn = dataSource.getConnection()) {
    // use connection — automatically returned to pool when closed!
}

// Shutdown pool when app exits
dataSource.close();

Top Interview Questions & Model Answers
Q1. What is JDBC? What are its key components?
JDBC (Java Database Connectivity) is a Java API in java.sql that allows Java programs to connect to relational databases. Key components: Connection (represents a database session), Statement (executes static SQL), PreparedStatement (precompiled SQL with parameters — prevents SQL injection), ResultSet (holds query results), and DriverManager (manages database drivers and creates connections).
Q2. What is the difference between Statement and PreparedStatement?
Statement executes static SQL strings — simple but vulnerable to SQL injection and recompiled every time. PreparedStatement precompiles the SQL once with placeholders (?), then accepts parameters at runtime — prevents SQL injection (parameters are treated as literals, not SQL), faster for repeated execution (compiled once), and cleaner code. Always prefer PreparedStatement when user input is involved.
Q3. What is SQL Injection? How does PreparedStatement prevent it?
SQL Injection is a security attack where malicious SQL code is inserted into an input field to manipulate database queries. Example: entering ' OR '1'='1 as a username can bypass authentication. PreparedStatement prevents this by separating SQL structure from data — the query structure is fixed at preparation time, and user input is always treated as a literal string value, never as SQL code to execute.
Q4. What is a JDBC Transaction? Why is it important?
A transaction is a group of SQL operations that must all succeed or all fail together (atomicity). Example: a bank transfer needs to debit one account AND credit another — if one fails, both must be rolled back. Without transactions, partial updates leave data inconsistent. By default, JDBC auto-commits each statement. For transactions: setAutoCommit(false), execute operations, then commit() on success or rollback() on failure.
Q5. What is Connection Pooling? Why is it needed?
Creating a database connection is expensive — it involves network handshake, authentication, and resource allocation. In web applications handling many requests, creating a new connection per request would be too slow. Connection pooling maintains a pre-created pool of connections that are borrowed and returned. This dramatically improves performance. HikariCP is the most popular pool in Java (default in Spring Boot).
Q6. What is ResultSet? How do you iterate it?
ResultSet is an object that holds the results of a SELECT query as a table. The cursor starts before the first row. Call next() to advance row by row — returns true if a row exists, false when exhausted. Use getXxx() methods like getString("columnName"), getInt("id") to retrieve values by column name (preferred) or column index (1-based). Always close ResultSet, Statement, and Connection in reverse order (or use try-with-resources).
Q7. What is the difference between executeQuery(), executeUpdate(), and execute()?
executeQuery() is for SELECT statements — returns a ResultSet. executeUpdate() is for INSERT, UPDATE, DELETE, and DDL (CREATE, DROP) — returns int (number of rows affected). execute() can run any SQL — returns boolean (true if result is a ResultSet, false if update count). Use the specific method when you know the statement type; use execute() for dynamic SQL where you don't know the type ahead of time.
