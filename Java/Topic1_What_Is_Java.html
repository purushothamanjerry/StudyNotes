What is Java? — One-Line Definition
Interview-ready definition:
Java is a high-level, object-oriented, platform-independent programming language developed by Sun Microsystems in 1995. It follows the principle of "Write Once, Run Anywhere" (WORA) — Java code compiles to bytecode that runs on any machine with a JVM installed.

🔑 Key Words to Remember

  • High-level — close to human language, not machine code
  • Object-Oriented — everything is modeled as objects and classes
  • Platform-Independent — runs on any OS without recompiling
  • Compiled + Interpreted — compiled to bytecode, then interpreted by JVM
  • Strongly Typed — variable types must be declared
  • Garbage Collected — automatic memory management

History — Who, When, Where

Who created Java?

James Gosling — known as the "Father of Java"

Team: Mike Sheridan, Patrick Naughton at Sun Microsystems

Where & When?

Sun Microsystems, California, USA

Project started: 1991 (called "Oak")

Publicly released: May 23, 1995

📅 Timeline

1991
James Gosling starts "Green Project" at Sun Microsystems. Language originally named Oak (after a tree outside Gosling's window). Goal: language for embedded systems (set-top boxes, TVs).
1995
Renamed to Java (inspired by Java coffee). Officially released as Java 1.0. Famous slogan: "Write Once, Run Anywhere". Integrated with Netscape browser — Java applets became popular.
2006
Sun Microsystems releases Java as open-source software under GNU GPL.
2010
Oracle Corporation acquires Sun Microsystems and takes ownership of Java.
2017
Java moves to a 6-month release cycle (new version every 6 months). LTS (Long-Term Support) versions every 3 years.
2025
Java 24 released. Java remains the #1 or #2 most used language globally (TIOBE index).
💡 Memory Tip: "James Gosling → Oak (1991) → Java (1995) → Sun Microsystems → Oracle (2010)"

JDK vs JRE vs JVM — The Most Asked Question
Golden Rule: JDK ⊃ JRE ⊃ JVM  (JDK contains JRE, JRE contains JVM)

JVM — Java Virtual Machine

JVM is an abstract machine — software that provides a runtime environment to execute Java bytecode. It is platform-specific (different JVM for Windows, Linux, Mac) — this is what makes Java platform-independent.

  • Converts bytecode → machine code using the JIT compiler
  • Manages memory (heap, stack, method area)
  • Performs garbage collection
  • Handles exception handling at runtime
  • Does bytecode verification for security
⚠️ Common Mistake: JVM is NOT platform-independent. The bytecode is platform-independent. JVM itself is platform-specific — there's a different JVM for each OS.

JRE — Java Runtime Environment

JRE = JVM + Libraries needed to run Java programs. If you only want to run Java programs (not write them), install JRE.

  • Includes: JVM + core class libraries (java.lang, java.util, etc.)
  • Does NOT include: compiler (javac), debugger, development tools
  • Used by: end users who just want to run Java applications

JDK — Java Development Kit

JDK = JRE + Development Tools. If you want to write and compile Java programs, install JDK.

  • Includes everything in JRE plus:
  • javac — the Java compiler
  • java — the Java launcher (runs .class files)
  • javadoc — generates documentation
  • jar — creates .jar archive files
  • jdb — Java debugger

🔁 How They Work Together

// Step 1: You write Java source code
HelloWorld.java   ← .java file (human readable)

// Step 2: JDK's javac compiler compiles it
javac HelloWorld.java
HelloWorld.class  ← .class file (bytecode, NOT machine code)

// Step 3: JVM inside JRE executes the bytecode
java HelloWorld
// JVM reads bytecode → JIT compiles to native machine code → runs

First Java Program — Every Line Explained
public class HelloWorld {

    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }

}

Line-by-Line Breakdown

CodeWhat it means
public class HelloWorld public = accessible from anywhere. class = blueprint keyword. HelloWorld = class name. File name must match class name → save as HelloWorld.java
public static void main This is the entry point of every Java program. JVM looks for exactly this signature to start execution.
public = JVM can access it. static = JVM calls it without creating an object. void = returns nothing. main = name JVM looks for.
String[] args Command-line arguments passed when running: java HelloWorld arg1 arg2. Even if you don't use them, this parameter is required.
System.out.println System = built-in class. out = static field (a PrintStream object). println = method that prints and adds a new line.

Compilation & Running

# Compile (creates HelloWorld.class bytecode file)
javac HelloWorld.java

# Run (JVM executes the bytecode)
java HelloWorld

# Output:
Hello, World!
Why is main() static?
Because JVM needs to call main() without creating an object of the class first. If it were non-static, JVM would need to instantiate the class — but to do that, it would need to call the constructor — and to know which constructor to call, it needs the class structure — it becomes a chicken-and-egg problem. Making it static lets JVM call it directly on the class.

Top Interview Questions & Model Answers
Q1. What is Java? Why is it platform-independent?
Java is a high-level, object-oriented programming language released in 1995 by Sun Microsystems. It is platform-independent because Java source code is compiled into bytecode (not machine code). This bytecode is not specific to any OS. Any machine that has a JVM installed can execute this bytecode — so the same .class file runs on Windows, Linux, and Mac without any changes.
Q2. What is the difference between JDK, JRE, and JVM?
JVM (Java Virtual Machine) executes bytecode and provides runtime environment. It is platform-specific.
JRE (Java Runtime Environment) = JVM + core libraries. Used to run Java programs.
JDK (Java Development Kit) = JRE + development tools (javac, javadoc, jdb). Used to develop Java programs.
Think of it as: JDK ⊃ JRE ⊃ JVM.
Q3. Is Java purely object-oriented?
No. Java is not purely object-oriented because it has primitive data types (int, char, boolean, etc.) which are not objects. A purely OO language like Smalltalk treats everything as an object. Java added wrapper classes (Integer, Character) to treat primitives as objects when needed.
Q4. What is bytecode?
Bytecode is the intermediate representation of Java code generated by the javac compiler. It is stored in .class files. It is not machine code — it cannot run directly on hardware. The JVM reads bytecode and either interprets it or compiles it to native machine code via the JIT compiler at runtime. Bytecode is platform-neutral, which is the core of Java's WORA principle.
Q5. Why is the main method static in Java?
Because JVM needs to call main() before any object is created. If main() were non-static, JVM would need an object of the class to call it — but no objects exist yet at startup. Making it static allows JVM to invoke it directly on the class without instantiation.
Q6. What is JIT compiler? How does it help performance?
JIT (Just-In-Time) compiler is part of the JVM. It monitors the bytecode being executed and compiles frequently-used (hot) code into native machine code at runtime. This native code is cached and reused, making subsequent calls much faster. Without JIT, every line would be interpreted one at a time — much slower.
Q7. What is the difference between C++ and Java?
Main differences: Java has no pointers (safe), Java has automatic garbage collection (C++ needs manual memory management), Java is platform-independent (C++ is not), Java doesn't support multiple inheritance of classes (uses interfaces instead), Java has no delete keyword, and Java has no header files.
Q8. Who owns Java now?
Oracle Corporation. Oracle acquired Sun Microsystems in 2010 and took over Java. The OpenJDK (open-source JDK) is maintained by Oracle and the community under the GPL license.
