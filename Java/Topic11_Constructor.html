What is a Constructor?
Definition: A constructor is a special method that is automatically called when an object is created using the new keyword. Its purpose is to initialize the object's state (set initial values for fields).

Why constructors?

  • Guarantee that an object is in a valid state when created
  • Avoid having to manually set fields after creation
  • Enforce mandatory fields (must provide name, age, etc.)
  • Set up resources (database connections, file handles)
class Car {
    String brand;
    int    year;

    // Constructor — same name as class, no return type
    Car(String brand, int year) {
        this.brand = brand;
        this.year  = year;
    }
}

// Constructor called automatically when new is used
Car c = new Car("Toyota", 2023);
// c.brand = "Toyota", c.year = 2023 — already initialized!

3 Types of Constructors
1. Default Constructor

No parameters. Provided by Java if no constructor defined.

2. Parameterized Constructor

Takes parameters to initialize fields with custom values.

3. Copy Constructor

Takes another object of same class and copies its values.

4
Default Constructor
Two meanings of "default constructor":
1. A constructor with no parameters (no-arg constructor)
2. The constructor automatically provided by Java if you write no constructor at all 
// Case 1: Java provides default constructor automatically
class Dog {
    String name;   // default: null
    int age;        // default: 0
    // No constructor written — Java adds this behind the scenes:
    // Dog() { super(); }  ← calls Object constructor
}
Dog d = new Dog();   // ✅ works! uses Java's default constructor
System.out.println(d.name);  // null
System.out.println(d.age);   // 0

// Case 2: You write a no-arg constructor explicitly
class Cat {
    String name;
    int age;

    // Explicit no-arg constructor with defaults
    Cat() {
        this.name = "Unknown";
        this.age  = 1;
    }
}
Cat c = new Cat();
System.out.println(c.name);  // Unknown
⚠️ Java's default constructor disappears if you add ANY constructor!
If you define even ONE parameterized constructor, Java stops providing the default.
new Dog() → ❌ compile error if Dog has only a parameterized constructor and no no-arg constructor!
Solution: explicitly add a no-arg constructor if needed.
5
Parameterized Constructor

Takes parameters to set field values at the time of object creation. Most commonly used constructor type.

class Student {
    private String name;
    private int    age;
    private double gpa;

    // Parameterized constructor
    public Student(String name, int age, double gpa) {
        this.name = name;
        this.age  = age;
        this.gpa  = gpa;
    }

    public void display() {
        System.out.println(name + " | " + age + " | " + gpa);
    }
}

// Object created with values immediately
Student s1 = new Student("Alice", 20, 3.8);
Student s2 = new Student("Bob",   22, 3.5);
s1.display();  // Alice | 20 | 3.8
s2.display();  // Bob   | 22 | 3.5
6
Copy Constructor

A constructor that takes an object of the same class as parameter and copies its field values. Java does NOT provide a copy constructor automatically (unlike C++) — you must write it.

class Student {
    String name;
    int    age;

    // Regular constructor
    Student(String name, int age) {
        this.name = name;
        this.age  = age;
    }

    // Copy constructor — takes object of same class
    Student(Student other) {
        this.name = other.name;
        this.age  = other.age;
    }
}

Student s1 = new Student("Alice", 20);
Student s2 = new Student(s1);   // copy constructor

s2.name = "Bob";   // changes only s2
System.out.println(s1.name);  // Alice — unchanged
System.out.println(s2.name);  // Bob

Copy Constructor vs Assignment

Student s2 = s1; → copies the reference — both point to same object

Student s2 = new Student(s1); → creates a new independent object

Shallow vs Deep Copy

Shallow — copies primitive values + references (not objects they point to)

Deep — also copies the objects that reference fields point to (fully independent)

Constructor Overloading

Having multiple constructors in the same class with different parameters. Gives flexibility in how objects are created.

class Rectangle {
    double width;
    double height;

    // No-arg — default square of 1x1
    Rectangle() {
        this.width  = 1.0;
        this.height = 1.0;
    }

    // Square — both sides equal
    Rectangle(double side) {
        this.width  = side;
        this.height = side;
    }

    // Full — specify both dimensions
    Rectangle(double width, double height) {
        this.width  = width;
        this.height = height;
    }

    double area() { return width * height; }
}

Rectangle r1 = new Rectangle();          // 1x1
Rectangle r2 = new Rectangle(5.0);       // 5x5
Rectangle r3 = new Rectangle(4.0, 6.0);  // 4x6

Constructor Chaining — this() and super()
Constructor chaining = calling one constructor from another constructor.
this() — calls another constructor in the same class
super() — calls a constructor in the parent class
Both MUST be the first statement in the constructor body!

this() — Same Class Chaining

class Employee {
    String name;
    int    age;
    String dept;

    // Full constructor
    Employee(String name, int age, String dept) {
        this.name = name;
        this.age  = age;
        this.dept = dept;
    }

    // Calls full constructor with default dept
    Employee(String name, int age) {
        this(name, age, "General");  // ← must be FIRST line
    }

    // Calls 2-param constructor with default age
    Employee(String name) {
        this(name, 25);  // ← must be FIRST line
    }
}

Employee e1 = new Employee("Alice");
// Chain: Employee(String) → Employee(String,int) → Employee(String,int,String)
// Result: name="Alice", age=25, dept="General"

super() — Parent Class Chaining

class Animal {
    String name;

    Animal(String name) {
        this.name = name;
        System.out.println("Animal: " + name);
    }
}

class Dog extends Animal {
    String breed;

    Dog(String name, String breed) {
        super(name);   // ← calls Animal constructor FIRST
        this.breed = breed;
        System.out.println("Dog breed: " + breed);
    }
}

Dog d = new Dog("Bruno", "Labrador");
// Output:
// Animal: Bruno
// Dog breed: Labrador
Important: Java automatically inserts super() as the first line of EVERY constructor if you don't write it — calling the parent's no-arg constructor. If the parent has no no-arg constructor, you MUST explicitly call super(params).
⚠️ You cannot use both this() and super() in the same constructor!
Both must be the first statement — so you can only use one. Using both causes a compile error.

Top Interview Questions & Model Answers
Q1. What is a constructor? How is it different from a method?
A constructor is a special block of code that is automatically called when an object is created using new. It has the same name as the class and no return type. Its purpose is to initialize the object's state. A method is a regular block of code with a name (different from class) and a return type, called explicitly by the programmer to perform operations.
Q2. What is a default constructor? When does Java provide it?
A default constructor is a no-argument constructor. Java automatically provides one if and only if you do NOT define any constructor in the class. The moment you define even one constructor (with or without parameters), Java stops providing the default. The auto-generated default constructor simply calls super() to initialize the parent class.
Q3. What is constructor overloading?
Constructor overloading is having multiple constructors in the same class with different parameter lists. This gives flexibility in how objects are created — with all fields, with some fields, or with no fields. The compiler determines which constructor to call based on the number and types of arguments passed with new.
Q4. What is constructor chaining? What is this() and super()?
Constructor chaining is calling one constructor from another. this() calls another constructor in the same class — useful to avoid duplicate initialization code. super() calls a constructor in the parent class — used to initialize inherited fields. Both must be the FIRST statement in a constructor and cannot be used together in the same constructor.
Q5. Can a constructor be private? When would you use it?
Yes. A private constructor prevents object creation from outside the class. This is used in: (1) Singleton pattern — ensure only one instance exists. (2) Utility classes — classes with only static methods (like Math) where you don't want instances. (3) Factory method pattern — control object creation through static factory methods.
Q6. Can a constructor return a value?
No. A constructor cannot have a return type — not even void. If you add a return type, it becomes a regular method, not a constructor. However, the constructor implicitly returns the newly created object when called via new.
Q7. What happens if the parent class has no default constructor?
If the parent class has only a parameterized constructor (no no-arg constructor), then every child class constructor MUST explicitly call super(params) as the first statement. If not, the compiler will try to call the non-existent super() and give a compile error: "constructor Animal() is undefined".
