What is a Method?
Definition: A method is a named block of code that performs a specific task. It is defined once and can be called (executed) multiple times from anywhere in the program.

Why use methods?

  • Reusability — write once, use many times
  • Modularity — break large problems into small pieces
  • Readability — code is easier to understand
  • Maintainability — fix in one place, fixed everywhere
  • Abstraction — hide implementation details

4
return Statement

The return statement does two things: returns a value to the caller and immediately exits the method.

// return exits method immediately
public int findFirst(int[] arr, int target) {
    for (int i = 0; i < arr.length; i++) {
        if (arr[i] == target) {
            return i;   // exits immediately when found
        }
    }
    return -1;   // not found
}

// void method can use return to exit early
public void printPositive(int n) {
    if (n <= 0) {
        System.out.println("Not positive!");
        return;   // exits early, no value returned
    }
    System.out.println(n);
}

// Multiple return points — valid but use carefully
public String grade(int score) {
    if (score >= 90) return "A";
    if (score >= 70) return "B";
    if (score >= 50) return "C";
    return "F";
}
⚠️ Every non-void method MUST have a return statement that covers all code paths. If the compiler sees a path where no value is returned → compile error: "missing return statement".

Method Overloading
Definition: Having multiple methods with the same name in the same class, but with different parameters (different number, type, or order of parameters). Resolved at compile time (static polymorphism). 
public class Calculator {

    // Different number of parameters
    public int add(int a, int b) {
        return a + b;
    }

    public int add(int a, int b, int c) {
        return a + b + c;
    }

    // Different parameter types
    public double add(double a, double b) {
        return a + b;
    }

    // Different order of types
    public void display(String s, int n) { }
    public void display(int n, String s) { }
}

Calculator c = new Calculator();
c.add(2, 3);       // calls first  → 5
c.add(2, 3, 4);    // calls second → 9
c.add(2.0, 3.0);  // calls third  → 5.0
⚠️ Return type alone CANNOT overload a method!
public int add(int a, int b) and public double add(int a, int b) → ❌ Compile error!
The compiler cannot distinguish methods by return type alone.
Valid OverloadingNot Valid
Different number of parametersOnly return type is different
Different parameter typesOnly parameter names are different
Different order of parameter typesOnly access modifier is different

Recursion
Definition: A method that calls itself. Every recursive method must have a base case (stopping condition) to prevent infinite recursion.

Every recursive solution has 2 parts:

  • Base case — condition to stop recursion (prevents infinite loop)
  • Recursive case — method calls itself with a smaller problem
// Factorial — n! = n × (n-1) × (n-2) × ... × 1
public int factorial(int n) {
    // Base case — stop here
    if (n == 0 || n == 1) return 1;

    // Recursive case — calls itself
    return n * factorial(n - 1);
}

// factorial(4) trace:
// factorial(4) = 4 × factorial(3)
//                    = 3 × factorial(2)
//                         = 2 × factorial(1)
//                              = 1 (base case)
// = 4 × 3 × 2 × 1 = 24

// Fibonacci — 0,1,1,2,3,5,8,13...
public int fibonacci(int n) {
    if (n <= 1) return n;  // base case
    return fibonacci(n - 1) + fibonacci(n - 2);
}
⚠️ No base case = StackOverflowError!
Without a base case, the method calls itself forever until the call stack is full → StackOverflowError

Recursion Pros

  • Elegant, clean code
  • Natural for tree/graph problems
  • Easier to understand logic

Recursion Cons

  • Uses more stack memory
  • Can be slower than loops
  • Risk of StackOverflowError

Varargs (Variable Arguments)

Varargs allows a method to accept any number of arguments of the same type. Internally treated as an array.

// Syntax: type... paramName
public int sum(int... numbers) {
    int total = 0;
    for (int n : numbers) {
        total += n;
    }
    return total;
}

// Call with any number of arguments
sum(1, 2);              // 3
sum(1, 2, 3);           // 6
sum(1, 2, 3, 4, 5);    // 15
sum();                    // 0 (zero arguments)
⚠️ Varargs rules:
❌ Only ONE varargs parameter per method
❌ Varargs must be the LAST parameter
✅ void log(String prefix, int... values) — valid
❌ void log(int... a, int... b) — invalid

Static vs Instance Methods
public class MathUtils {

    // Static method — belongs to CLASS
    public static int square(int n) {
        return n * n;
    }

    // Instance method — belongs to OBJECT
    private int value;

    public void setValue(int v) {
        this.value = v;
    }

    public int getValue() {
        return value;
    }
}

// Static method — call via CLASS NAME (no object needed)
int s = MathUtils.square(5);    // 25

// Instance method — need to create OBJECT first
MathUtils m = new MathUtils();
m.setValue(10);
System.out.println(m.getValue());  // 10

Top Interview Questions & Model Answers
Q1. What is method overloading? How is it different from overriding?
Method overloading is having multiple methods with the same name but different parameters in the SAME class. It is resolved at compile time (static polymorphism). Method overriding is redefining a parent class method in a child class with the same name and parameters. It is resolved at runtime (dynamic polymorphism). Overloading = same class, different params. Overriding = parent-child, same params.
Q2. Can you overload a method by changing only the return type?
No. Return type alone is NOT enough to overload a method. The compiler identifies methods by their name and parameter list (method signature) — not by return type. Having two methods with the same name and parameters but different return types causes a compile error.
Q3. What is the difference between parameter and argument?
A parameter is the variable defined in the method declaration — it acts as a placeholder. An argument is the actual value passed when calling the method. Example: in void greet(String name), name is the parameter. In greet("Alice"), "Alice" is the argument.
Q4. What is recursion? What happens if there is no base case?
Recursion is when a method calls itself to solve a smaller version of the same problem. Every recursive method needs a base case — a stopping condition that returns without making another recursive call. Without a base case, the method calls itself infinitely until the call stack runs out of space, causing a StackOverflowError.
Q5. Can a static method access instance variables? Why?
No. Static methods belong to the class, not to any specific object. Instance variables belong to objects. When a static method runs, there may be no object (instance) created at all — so there is no this reference and no instance variable to access. Static methods can only access other static members directly.
Q6. What are varargs? Write an example.
Varargs (variable arguments) allows a method to accept any number of arguments of the same type using the ... syntax. Example: public int sum(int... numbers) can be called as sum(1,2), sum(1,2,3,4), or even sum(). Internally, varargs is an array. Rules: only one varargs per method, and it must be the last parameter.
Q7. What is the output? And why? 
public static int factorial(int n) {
    if (n == 1) return 1;
    return n * factorial(n - 1);
}
System.out.println(factorial(4));
24 — factorial(4) = 4 × factorial(3) = 4 × 3 × factorial(2) = 4 × 3 × 2 × factorial(1) = 4 × 3 × 2 × 1 = 24. The base case n == 1 stops the recursion.
